# AI能力分析

> 负责人：成员D

## 目标

明确AI需要具备哪些能力，调研和选择合适的技术方案。

## 输出清单

- [ ] [AI能力定义](./capabilities.md)
- [ ] [技术调研](./tech-research.md)
- [ ] [知识库规划](./knowledge-base.md)
- [ ] [Prompt设计](./prompt-design.md)

## 关注点

1. **AI能力定义**
   - 系统需要AI具备哪些能力
   - 每种能力的边界（能做到什么程度，做不到什么）
   - 对用户体验的影响

2. **技术调研**（重点）
   - 大模型选择（建议实际测试）
   - PDF/视频/图片处理方案
   - 向量数据库和RAG方案
   - PPT/Word生成方案
   - 对比各方案的优劣

3. **知识库规划**
   - 知识库需要包含哪些内容
   - 如何组织和检索
   - 数据格式和处理

4. **Prompt设计**
   - 核心场景的Prompt设计思路
   - 如何引导模型生成符合要求的内容
   - 多轮对话的上下文管理

## 待调研的技术

参考 [技术栈文档](../../01-PROJECT/tech-stack.md#待调研的技术需求分析阶段)

### 必须调研
- 大语言模型选型
- 文档解析方案
- 向量数据库选型
- PPT/Word生成方案

### 建议调研
- Embedding模型
- 视频处理方案
- 图片OCR方案
- 模型路由方案

## 建议

- **实际测试**：不要只看文档，实际测试候选方案
- **对比分析**：列出各方案的优劣、成本、性能
- **给出建议**：基于测试结果给出明确的技术选型建议
- **考虑成本**：评估API调用成本、开发成本、维护成本

## 工具推荐

- Jupyter Notebook（测试代码）
- Postman（测试API）
- 各大模型的Playground

创建以下文件并开始分析：
- `capabilities.md` - AI能力定义
- `tech-research.md` - 技术调研（重点）
- `knowledge-base.md` - 知识库规划
- `prompt-design.md` - Prompt设计
