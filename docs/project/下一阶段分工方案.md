# 下一阶段分工方案

> 更新时间：2026-03-22
> 状态：生效中
> 适用阶段：初审准备阶段

## 1. 文档目的

本文档用于明确 Spectra 在下一阶段的工作重点与 ABCD 分工。

当前阶段的核心目标不是继续扩展系统边界，而是：

1. 对齐竞赛原题
2. 稳定主链路
3. 收束项目亮点
4. 准备可用于初审的项目材料

## 2. 当前项目判断

基于当前代码、文档和评测资产，项目状态可以概括为：

- 已明显超出普通竞赛 demo 水平
- 已具备较完整的产品闭环和后端架构
- 已具备多模态资料处理、对话理解、PPT/Word 生成、预览修改、导出下载等核心能力
- 已具备较强的技术与工程基础，包括 OpenAPI 契约、测试、评测、基线、状态与事件语义

当前主要问题不是“功能不够”，而是：

- 初审叙事不够收束
- 主链路需要进一步稳定
- 技术亮点需要转化为更容易理解的证据

## 3. 下一阶段的总原则

下一阶段按以下原则执行：

1. 以“先过初审”为第一目标。
2. 以题目要求为主，不以系统扩张为主。
3. 以主链路稳定优先于边缘功能扩展。
4. 以可展示、可截图、可写进材料的成果优先。
5. 以低冲突、低返工优先于高并行。

## 4. 下一阶段重点

下一阶段只围绕四件事推进：

### 4.1 题目对齐

要明确回答以下问题：

- 我们是否满足“理解意图”
- 我们是否满足“融合多模态参考”
- 我们是否满足“生成 PPT/Word”
- 我们是否满足“预览、修改、再生成”

### 4.2 主链路稳定

主链路统一为：

`上传资料 -> 对话澄清 -> 生成大纲/内容 -> 生成 PPT/Word -> 预览 -> 修改 -> 导出`

这条链路必须稳定、连贯、可演示。

### 4.3 亮点收束

下一阶段只保留三类对外亮点：

1. 多模态资料融合
2. 会话式生成与迭代优化闭环
3. 可验证的技术质量能力（如检索、引用、记忆一致性）

### 4.4 材料准备

初审所需材料应围绕以下内容准备：

- 题目要求对照表
- 主链路流程图
- 系统架构图
- 页面截图和结果样例
- 技术亮点证据

## 5. ABCD 分工

### 5.1 A：总控与材料收口

A 负责：

- 明确项目主叙事
- 明确题目要求对照关系
- 决定材料写什么、不写什么
- 决定下一阶段只保留哪些亮点
- 收口前后端、技术、材料三边语言
- 在 C/D 发生冲突时做最终判断

A 当前阶段不默认承担：

- C 的主线实现
- D 的专题实现
- 前端页面细节补位

A 的产出应包括：

- 一版题目要求对照表
- 一版项目简介结构
- 一版项目亮点清单
- 一版初审材料目录

### 5.2 B：展示与材料可见层

B 负责：

- 主链路页面整理
- 上传、对话、生成、预览、下载这条链路的展示完整性
- 截图、录屏和界面展示素材
- 页面层面的演示路径优化
- 如保留卡片能力，只保留最适合初审展示的 1 至 2 个

B 当前阶段不优先承担：

- 大量新页面扩展
- 与初审无关的界面探索
- 反向定义后端语义

B 的目标是：

- 让评委一眼看出“项目已经做成了”

### 5.3 C：主线稳定与交付

C 负责：

- 上传、生成、预览、导出、下载等主链路稳定
- 会话、状态、事件、产物语义一致性
- 主线接口稳定性
- 生成结果落到项目空间产物的闭环
- 与 B 对接所需的稳定接口

C 当前阶段不优先承担：

- D 的技术专题
- 没有明确收益的探索性改动
- 边缘功能扩展

C 的目标是：

- 保证主链路可用、可演示、可写材料

### 5.4 D：技术亮点证据化

D 负责：

- 从现有专题中只选 1 至 2 个最有价值的方向
- 输出可量化的基线、改进结果和结论
- 给出可写入初审材料的技术说明
- 必要时给出接入主线的建议，但不默认接管主线

D 当前阶段建议只做以下两个方向中的一到两个：

1. 检索与重排质量
2. 对话记忆与引用一致性

如有余力，再处理：

3. 上传链路降级信息统一

D 当前阶段不优先承担：

- 新方向发散
- 零散业务补洞
- 无基线的主观优化

D 的目标是：

- 把“技术深度”转化成“初审材料里的可信证据”

## 6. 协作方式

下一阶段默认采用以下协作方式：

1. 主线工作默认由 C 推进。
2. 技术专题默认由 D 推进。
3. 需要同时修改同一块关键逻辑时，默认拆成前后阶段串行。
4. 语义、契约、优先级变化由 A 判断。
5. B 不等待不稳定接口，优先基于已定路径推进展示层。

## 7. 本阶段不建议优先做的事

以下内容当前阶段不建议优先推进：

- 大规模扩展新卡片能力
- 新的抽象系统设计
- 与初审无直接关系的深度重构
- 无法形成展示价值的底层优化
- 没有明确证据产出的开放式研究

## 8. 阶段完成标准

如果下一阶段执行有效，应达到以下结果：

1. 有一份清晰的题目要求对照表。
2. 有一条稳定的主链路可以完整演示。
3. 有一套页面截图和结果样例可直接进入材料。
4. 有 1 至 2 项技术亮点具备可量化证据。
5. ABCD 各自知道本阶段的主要责任，不再反复重定义分工。

## 9. 一句话总结

下一阶段不是继续把系统做大，而是把已有能力收束成一个强对题、强完成度、强证据的初审项目。
