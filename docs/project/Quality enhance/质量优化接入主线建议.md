# 质量优化接入主线建议

> 更新时间：2026-03-25
> 状态：当前生效
> 定位：成员 D 面向主线接入的质量优化建议

## 1. 目标

这份建议用于回答一个具体问题：

1. D 侧已经形成的质量规则、样本、baseline 和结论，应该如何稳定接入主线
2. 如何避免质量优化变成新的隐藏分叉
3. 如何让 C/B/A 能直接消费这些结果

当前原则很明确：

1. 主链路功能补完归 C
2. 质量优化归 D
3. D 的输出必须接入主线，而不是平行长出第二套逻辑

## 2. 默认接入原则

质量优化默认按以下原则接入：

1. 先进入主线的统一 prompt / service / baseline 入口
2. 不新增平行语义和临时旁路
3. 先有样本和 baseline，再推动规则上线
4. 上线后必须能被现有 audit / baseline / gate 观测到

这意味着，D 的输出不能以“单独脚本跑通”为终点。

## 3. 哪些内容应该直接接入主线

### 3.1 提示词规则

以下类型的优化，默认应直接落到主线提示词模块：

1. PPT 页面结构约束
2. 信息密度与图文比例约束
3. 图片插入与素材命中规则
4. 大纲去重、避免标题和知识点复述的规则

当前对应入口已经存在：

1. `backend/services/prompt_service/`
2. `backend/services/courseware_ai/`

接入要求：

1. 不在局部脚本里复制一份 prompt
2. 不在调试分支里保留“只对某个场景生效”的隐藏 prompt 分支

### 3.2 baseline 与 gate

以下类型的结果，默认应直接纳入现有评测与门禁链路：

1. PPT 质量样本与 baseline
2. 大纲重复质量 baseline
3. 检索、来源、网络资源相关质量证据
4. Project Space 最终 quality gate

接入要求：

1. 优化结果必须能回到已有 `eval/` 目录工具链
2. 不再新增一套平行的“临时评测表”

### 3.3 文档级方法与初审证据

以下类型的结果，默认应直接进入项目文档层：

1. 质量方法
2. 质量证据
3. 主线接入建议
4. 可复用的技术结论

目的是让 A/E 可以直接复用，而不是再二次解释 D 在做什么。

## 4. 哪些内容不应直接落主线

以下内容不应由 D 直接当作主线代码落地：

1. 真正 `.pptx / .docx` 产物闭环能力本身
2. 主链路 artifact / download / export 的功能补完
3. 卡片“从没有到有”的功能实现

原因：

1. 这些是 C 的职责
2. D 直接代做会导致职责边界失真
3. 容易形成“质量规则顺手带出一套新主链路”的隐藏分叉

## 5. 推荐接入顺序

### 第一层：规则先接入

先把已经验证过的质量规则接入现有主线入口：

1. prompt 规则
2. outline 去重规则
3. 图片命中与插图策略

### 第二层：样本和 baseline 跟上

规则接入后，必须立刻补：

1. 样本
2. baseline
3. 前后对照
4. guardrail

### 第三层：前端与演示消费

当质量规则和 baseline 进入稳定态后，再由 B/C 消费：

1. 展示更好的默认结果
2. 在前端和演示中体现“结果确实更稳”
3. 在初审材料里复用结论

## 6. 对 C 的建议

1. 优先消费 D 已经验证过的 prompt 与质量规则，不要另起一套本地修补逻辑
2. 主链路接入时，尽量走现有 `generation_session_service / prompt_service / courseware_ai` 入口
3. 接入后继续使用已有 baseline/check，避免“接进去以后无法验证”
4. 当功能补完与质量优化冲突时，优先保留主线语义一致性，再由 D 做二轮质量收口

## 7. 对 B 的建议

1. 前端演示默认优先消费已经通过 baseline/gate 的能力
2. 不把“偶然看起来不错”的单次结果直接当作演示样本
3. 对来源、图片、卡片结果的展示，尽量保留可解释线索，方便对应 D 的证据文档

## 8. 对 A / E 的建议

1. 对外材料优先使用“有样本、有 baseline、有对照”的结论
2. 不把未冻结的联调结果写成最终定论
3. 优先使用 D 已经整理好的质量证据和技术结论，减少二次转述失真

## 9. 接入完成的判断标准

一项质量优化只有同时满足以下条件，才算稳定接入主线：

1. 已落到现有主线入口，而不是旁路脚本
2. 有对应样本或 baseline
3. 能被现有 audit / check / gate 观察到
4. 不引入新的语义分叉
5. 能被 B/C/A/E 直接消费

## 10. 当前最适合先接入的内容

按当前完成度，最适合优先接入主线的是：

1. PPT 结构、密度、图文比例相关 prompt 规则
2. 图片命中与插图策略
3. 大纲去重规则
4. 卡片质量方法中的优先级和评估维度
5. 检索 / 来源 / 资源排序的证据口径
