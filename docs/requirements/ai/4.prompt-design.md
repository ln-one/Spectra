

### 4. Prompt 设计思路：教学逻辑的数字化翻译

#### 4.1 核心设计原则：COT + Role-Play + Structured Output

为了确保 Qwen-3.5 或 GPT-5 能够稳定产出高质量内容，Spectra 采用以下设计准则：

1. **角色注入（Persona Injection）**：
   
   * **提示词**：“你是一位拥有20年教研经验的‘高级学科教学设计师’和‘多模态内容专家’。你擅长运用 **BOPPPS** 模型设计课程，并能将枯燥的教材内容转化为生动、符合视觉传播规律的 PPT 脚本。”
   * **目的**：设定专业基调，确保生成内容的教学严谨性。

2. **思维链引导（Chain of Thought, CoT）**：
   
   * **提示词**：“在输出最终结果前，请先进行内部思考（Thinking Process）：1. 分析用户意图的学科范围；2. 提取检索到的知识库核心点；3. 规划本节课的教学逻辑流。请在 `<thought>` 标签中展示你的思考过程。”
   * **目的**：减少幻觉，提高复杂教学逻辑的准确度。

3. **强制结构化输出（Structured Output）**：
   
   * **提示词**：“请严格按照 JSON 格式返回结果，禁止输出任何无关的解释文字。JSON 结构必须包含：`slide_id`, `layout_type`, `content_structure`, `visual_prompt`。”
   * **目的**：自研方案的后端脚本需要解析 JSON 来生成 PPT/Word，结构化输出是实现“全自动生成”的前提。

---

#### 4.2 核心场景提示词模版（Scenario-Based Prompts）

##### 场景一：教学意图解析与大纲生成（Intent Parsing）

这是 Spectra 的第一步，将老师的碎语和上传的 PDF 转化为课程结构。

* **Prompt 思路**：
  * **输入**：用户的原始文本/语音 + 知识库检索片段。
  * **逻辑**：对比检索片段，识别知识点的层级（父点、子点、扩展点）。
  * **输出示例**：“请基于以下教材内容，生成一个 15 课时的‘教学进度表’。标注出哪些环节适合插入互动游戏，哪些环节需要播放视频素材。”

##### 场景二：多模态内容转化（PPT 脚本生成）

将解析后的文字转化为可供前端渲染的 PPT 逻辑。

* **Prompt 思路**：
  * **指令**：“为第 3 页 PPT‘光合作用的过程’设计内容。
  * **视觉指令**：请根据本页主题，生成一句给 DALL-E/Stable Diffusion 的绘图提示词，要求风格为‘扁平化科普插画’。
  * **内容约束**：正文不得超过 4 个要点，每点不超过 15 字，必须包含一个 LaTeX 格式的化学方程式。”

##### 场景三：互动互动游戏脚本化（Game Scripting）

响应赛题要求“至少支持一种互动小游戏”。

* **Prompt 思路**：
  * **指令**：“请根据本章‘牛顿运动定律’的重难点，设计一个‘课堂答题闯关’脚本。
  * **规则**：提供 3 个由易到难的选择题，并为每个错误选项提供一段幽默的 AI 解释。”

---

#### 4.3 迭代优化机制：增量式反馈提示词

自研方案最怕用户修改意见导致“全盘重推”。Spectra 采用 **“局部锁定”** 提示词技术。

* **设计思路**：
  * **上下文快照**：将当前页面的 JSON 数据传回 AI。
  * **差异指令**：“保持当前 PPT 第 5 页的标题和结构不变，仅根据用户的反馈‘图太复杂了，换成简单的示意图’，更新 `visual_prompt` 字段，并重新生成一张更简易的配图描述。”
  * **目的**：实现真正的“交互式备课”，让 AI 像真人助理一样听懂修改意见。

---

#### 4.4 负向约束与安全防护（Negative Constraints）

为了保证 Spectra 在比赛演示时的稳定性，必须在 System Prompt 中加入强约束：

* **内容约束**：
  * “禁止在课件中出现任何政治敏感、色情或暴力的描述。”
  * “如果知识库中没有相关信息，请明确告知用户‘知识库中未检索到确切依据，建议人工复核’，严禁编造数据。”
* **排版约束**：
  * “严禁在一页 PPT 中放置超过 300 字的文本。”
  * “所有的公式必须使用标准的 `$` 符号包裹的 LaTeX 语法。”

---

#### 4.5 核心思想：Prompt 动态注入

不使用静态提示词，而是采用动态拼接。系统会根据用户上传的文件类型（是视频还是 PDF），自动从我们的 **Prompt 库** 中选取最适合解析该格式的‘前置脚本’。例如，针对视频解析，我们会自动注入‘关注时间戳与动作描述’的特殊指令；针对 PDF 则是‘关注层级标题与公式溯源’。这种 **‘意图识别 -> 提示词路由 -> 结构化输出’** 的链路是 智能化的核心。”
