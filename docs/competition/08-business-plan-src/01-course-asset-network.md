# 01. 课程知识资产网络

## 要解决的商业问题

如果只把 `Spectra` 写成“AI 课件生成工具”，商业价值会被压缩到订阅费、生成额度和导出能力。这个口径太小，也不能解释为什么课程库、引用关系、版本演化和多模态外化值得长期建设。

第 8 章需要把商业本体改写为：`Spectra` 正在组织一个课程知识资产网络。文件只是外化结果，真正可持续经营的是课程库和围绕课程库形成的关系。

## 对应的产品/对象基础

商业对象必须回到系统对象：

- `Project / 课程库`：长期知识资产节点；
- `Session`：一次生产、修改、生成和外化的工作展开；
- `Artifact`：按需外化结果；
- `Version`：正式状态和交付版本锚点；
- `Reference`：课程库之间的引用、授权和影响力关系；
- `CandidateChange`：协作、改编和持续演化入口；
- `Member`：组织、成员和权限边界。

## 商业机制如何成立

课程库具备资产属性，是因为它不只保存内容，还保存可复用的关系：

- 可引用：其他课程库可以基于它继续生产；
- 可授权：不同成员、学习者、团队和机构可以获得不同权限；
- 可外化：同一课程库可以派生 `PPT`、教案、导图、动画、互动内容、说课材料和学情预演；
- 可演化：版本和候选变更让课程库不是一次性资源包；
- 可交易：高质量课程库可以进入订阅、授权、购买、分成和机构采购场景。

这使课程库从“内容容器”升级为“可经营资产”。

## 当前系统已经支撑什么

当前主稿可以稳妥表达：

- 系统已把 `Project / Session / Artifact / Version / Reference / CandidateChange / Member` 作为核心对象语言；
- 课程空间、`Studio` 多模态外化、artifact binding、版本/引用语义和成员边界已经构成商业资产叙事基础；
- 六个正式能力层支撑课程库生产、交付、检索、身份和状态管理；
- 当前阶段可以证明“课程库作为资产节点”的系统基础成立。

不要把“课程库交易市场已经完整上线”写成当前事实。交易、授权、收益分配是商业演进方向。

## 可进入第 8 章的外部化表述

`Project / 课程库` 不是普通文件夹，而是可引用、可授权、可外化、可演化、可交易的课程知识资产节点。一个优质课程库越被使用、引用和外化，它的长期商业价值就越高。

## 写作禁区与风险边界

- 不把课程库写成静态资源库。
- 不把导出文件写成系统本体。
- 不把未来交易能力写成当前已完整运营的 marketplace。
- 不写“平台拥有一切数据”这类容易引发数据合规风险的表述。
