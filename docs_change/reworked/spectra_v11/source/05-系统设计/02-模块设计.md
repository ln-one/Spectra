## 软件系统模块设计

### Studio 生成与执行模块

该模块负责围绕一次教学任务组织生成输入、确认中间结构、执行正式生成并承接后续修改。它对应当前前端工作台中的 Studio 区域。

![](../../assets/media/image2.png){width="5.1in" height="6.4in"}
图 5-4 Studio 生成与执行模块

在作品中，这一模块的页面入口是 Studio 生成配置、会话区域和卡片执行入口；接口入口包括 `POST /api/v1/generate/sessions`、`POST /api/v1/generate/studio-cards/{card_id}/execution-preview` 和 `POST /api/v1/generate/studio-cards/{card_id}/execute`；服务边界上，Spectra backend 负责会话编排与状态推进，`Diego` 负责 outline 与正式内容生成。结果层面，这一模块输出的是可以继续进入预览、导出和保存流程的正式结果对象。

这一模块把输入组织、方向确认、正式生成和后续修改拆成连续步骤。教师可以先看 `outline` 是否符合课程目标，再决定是否继续执行正式生成；生成结果出来后，也可以继续进入 refine 和预览流程。页面、接口和 focused tests 能够同时证明这部分能力已经进入当前实现。

从当前协议设计继续往下拆，Studio 也不只是“生成入口”。它还承担卡片目录、执行预览、正式执行以及 refine / turn / placement 这些后续动作的统一分流。对系统实现而言，当前至少存在两类主路径：一类是 `Session carrier`，典型代表是 `courseware_ppt`，先建立会话、推进 `outline` 与正式生成，再进入 preview/export；另一类是 `artifact carrier`，典型代表是 `word_document`、`interactive_quick_quiz`、`knowledge_mindmap` 等卡片，直接以 artifact create 进入正式结果链。这样写之后，Studio 模块就不再是抽象工具箱，而是能和当前 execution plan、页面入口和后续结果对象一一对应的正式编排层。

### 结果预览与导出模块

该模块负责把正式生成结果接入预览、历史查看和导出下载。它对应真实作品中的 preview 页面和导出入口。

![](../../assets/media/image4.png){width="7.0in" height="3.5in"}
图 5-5 结果预览与导出模块

页面入口包括结果预览页、slide 级查看页、历史查看和导出下载入口；接口入口包括 `GET /api/v1/generate/sessions/{session_id}/preview`、`GET /api/v1/generate/sessions/{session_id}/preview/slides/{slide_id}` 和 `POST /api/v1/generate/sessions/{session_id}/preview/export`；服务边界上，`Pagevra` 负责 preview / render / export，Spectra backend 负责结果绑定、版本关联和下载响应。用户看到的预览、历史查看与最终导出属于同一套结果。

这一模块保证两件事：一是预览来自正式结果；二是导出文件与预览页面保持一致。相关 preview contract tests 和 artifact 绑定测试可以直接复核这部分实现。

### 资料处理与检索模块

该模块负责资料上传后的解析、切块、索引、召回和来源查看。它对应当前作品里的资料区、上传入口和来源查看功能。

![](../../assets/media/image7.png){width="7.1in" height="3.2in"}
图 5-6 资料处理与检索模块

页面入口包括资料上传、来源查看和资料列表；接口入口包括 `POST /api/v1/files`、`POST /api/v1/rag/index`、`POST /api/v1/rag/search` 和 `GET /api/v1/rag/sources/{chunk_id}`；服务边界上，`Dualweave` 负责上传编排与远端解析入口，`Stratumind` 负责检索与向量召回，Spectra backend 负责请求组织、超时控制和响应整形。结果层面，这一模块的有效性由第 7 章正式评估表支撑。

这一模块决定上传资料能否真正进入后续生成流程。资料如果只停留在附件层，就无法支撑后续检索和来源查看；资料进入解析、索引和召回流程后，才会成为生成过程中可用的证据来源。

### 身份与成员边界模块

该模块负责用户身份、组织边界和成员权限。它对应系统设置、登录会话和成员边界控制。

页面入口主要体现在登录会话、系统设置和结果归属；接口与服务边界上，`Limora` 承接身份、组织和成员会话语义，Spectra backend 消费这些语义并作用到项目、资料和结果对象。它决定谁能进入某个工作过程、谁能看到某个结果、哪些结果可以继续被后续成员使用。

这四组模块直接对应当前作品中的四类功能：Studio 生成与执行、结果预览与导出、资料处理与检索、身份与成员边界。模块名、页面入口、接口入口和服务边界都可以在当前代码和作品界面中直接核对。
