# 提示词工程参考资料清单

> 更新时间：2026-03-27
> 状态：当前生效
> 定位：Spectra 当前阶段进行提示词优化时的参考资料入口

## 1. 作用范围

本清单不只服务 `PPT` 生成，也覆盖以下会被提示词工程显著影响的链路：

1. `Chat`
2. `RAG / 检索 / 引用`
3. `大纲生成`
4. `PPT / Word` 生成
5. `卡片生成`
6. `提示词评测 / 回归`

本清单是参考资料入口，不替代当前实现真相。

判断优先级仍然是：

1. 当前代码与测试
2. 当前 OpenAPI 与运行时行为
3. 当前项目文档
4. 本清单列出的参考资料

## 2. 使用原则

使用这些资料时，默认遵守以下规则：

1. 不直接抄通用模板到主链路。
2. 优先提炼结构，再落到 Spectra 的任务语义。
3. 不把 `Chat / RAG / 大纲 / PPT / 卡片 / revise` 混成一套 prompt。
4. 提示词优化必须尽量配套样本、对照和回归，不靠主观感觉判断。
5. 如果参考资料与当前实现冲突，以当前代码、测试和协作文档为准。

## 3. 最优先参考

### 3.1 OpenAI Prompt Engineering Guide

- 链接：https://platform.openai.com/docs/guides/prompt-engineering/strategies-to-improve-reliability
- 适合 Spectra 借鉴：
  - 把 prompt 固定为 `目标 + 输入材料 + 约束 + 输出格式`
  - 长任务分阶段，而不是一条 prompt 生成到底
  - few-shot 只在必要时加入
- 适合落点：
  - `backend/services/prompt_service/`
  - `backend/services/courseware_ai/`
  - `backend/routers/chat/`

### 3.2 OpenAI Best Practices for Prompt Engineering

- 链接：https://help.openai.com/en/articles/6654000-best-practices-for-prompt-engineering-with-the-openai-api
- 适合 Spectra 借鉴：
  - instruction 前置
  - 用分隔符明确区分任务和上下文
  - 先 zero-shot，再 few-shot，再考虑更重方案
- 适合落点：
  - 大纲生成和修改
  - PPT 页面结构约束
  - revise / refine prompt

### 3.3 Anthropic Prompt Engineering Overview

- 链接：https://docs.anthropic.com/en/docs/prompt-engineering
- 适合 Spectra 借鉴：
  - 什么时候该靠 prompt，什么时候该补系统结构
  - few-shot、结构化输出、长上下文控制
  - 用失败样本驱动 prompt 迭代
- 适合落点：
  - `RAG + 回答`
  - 资料利用与引用一致性
  - 卡片和大纲的长上下文输入组织

### 3.4 Anthropic Prompt Templates and Variables

- 链接：https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/prompt-templates-and-variables
- 适合 Spectra 借鉴：
  - 模板化、参数化 prompt
  - 避免每个模块自由拼接 prompt 文本
  - 复用时保持稳定变量接口
- 适合落点：
  - `prompt_service`
  - `courseware_ai`
  - `cards`
  - `refine` 流程

## 4. 评测与优化框架

### 4.1 DSPy

- 仓库：https://github.com/stanfordnlp/dspy
- 适合 Spectra 借鉴：
  - 把提示词优化从手工调参升级为样本驱动优化
  - 适合多步链路，不只适合单条 prompt
  - 用评价函数反推更优 prompt 结构
- 适合落点：
  - `chat -> retrieval -> answer`
  - `outline -> page plan -> content`
  - `card generation`

### 4.2 promptfoo

- 仓库：https://github.com/AI-App/PromptFoo
- 适合 Spectra 借鉴：
  - prompt A/B 对比
  - regression 检查
  - 提示词优化前后批量比较
- 适合落点：
  - `outline prompt`
  - `ppt prompt`
  - `card prompt`
  - `RAG answer prompt`

### 4.3 OpenAI simple-evals

- 仓库：https://github.com/openai/simple-evals
- 适合 Spectra 借鉴：
  - 用轻量脚手架做 prompt / model 对照实验
  - 不先引入复杂平台，也能跑稳定对比
- 适合落点：
  - `backend/eval/`
  - 新增 prompt 对照实验时的基础组织方式

### 4.4 Prompting GPT-3 To Be Reliable

- 论文：https://arxiv.org/abs/2210.09150
- 适合 Spectra 借鉴：
  - 不只看个别成功案例，要看泛化和稳定性
  - 关注 prompt wording 对输出稳定性的影响
- 适合落点：
  - 大纲生成稳定性
  - 卡片输出一致性
  - revise / refine prompt 的鲁棒性

## 8. Spectra 当前建议采用的四条原则

1. 先把 prompt 结构标准化，再谈局部 wording 微调。
2. 按任务拆模板，不混用 `chat / rag / outline / slide / card / revise`。
3. 提示词变更尽量配套样本、对照和回归，不靠主观判断。
4. 尽量让 prompt 先生成中间语义结构，再生成最终内容。
