# ADR-004: 大语言模型选择 - Qwen 3.5

**状态**: ✅ 已接受  
**日期**: 2026-02-19

## 背景

需要选择大语言模型作为系统的核心推理引擎，负责教学意图理解、对话交互、课件内容生成等任务。

## 考虑的方案

| 方案 | 优势 | 劣势 |
|------|------|------|
| **Qwen 3.5 (阿里)** | 中文教学术语理解极准，性价比高，响应快 | 开源工具生态不如 GPT |
| GPT-5 (OpenAI) | 全球广泛使用，开源工具多，多模态能力强 | 价格昂贵，需 Visa 支付 |
| Claude 3 | 长上下文支持好，安全性强 | 中文能力不如 Qwen |

## 决策

选择 **Qwen 3.5** 作为主控大脑

## 理由

### 实测结果
- 逻辑推理能力强，正确理解提示词并输出 PPT 大纲
- API 调用简单，响应迅速
- 中文排版和教学术语处理表现优异

### 成本优势
- API 价格约为 GPT 的 **1/10**
- 阿里云提供百万 Token 试用额度
- 支持国内支付方式

### 教育场景适配
- 中文教学语境理解精准
- 支持多模态扩展（图像理解）
- 可本地部署开源版本

## 权衡

- GPT 生态工具更丰富 → 但 Qwen 核心能力已足够
- 中文以外场景可能不如 GPT → 但本项目聚焦国内教育

## 影响

- 后端集成 Qwen API（通义千问）
- Prompt 设计针对 Qwen 特性优化
- 备选方案保留 GPT-4 接口
