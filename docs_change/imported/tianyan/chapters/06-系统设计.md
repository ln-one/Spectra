# 5.系统设计

## 5.1系统整体构架

本项目是基于人流车流的智能交通视频监控大数据可视化平台，包括监控设备、5G云、用户端、后端等部分，所述监控设备将采集到的实时监控视频传到5G云进行初步处理，然后发送到后端进行进一步处理，调用service和对应算法等成功对实时视频中的车辆速度、车流量、人流量和车牌信息等进行提取或者是违法车辆的跟踪报警等，在后端保存。当用户在PC端进行操作，后端从中拉取相应区域路口所含信息，在前端渲染并显示，实现数据可视化大屏的呈现。用户也可以在app端进行操作，通过与后端交互，实现同样效果。将视频处理得出的实时数据利用根据实测结果训练成功的岭回归模型得出车流量的预测，最后将预测数据导入所提出的 Optimal timing交通灯点亮算法，控制相应的合理交通灯配时，最终实现系统的智能导流功能。

系统关键技术方案将在下一章进行详细说明。

## 5.2软件系统

### 5.2.1可视化大屏

搭建可视化大数据监测平台web前端页面：用socket编译一个可拉取数据、视频、图表并进行渲染的后端，并用flash将拉取的信息呈现在web前端页面上。用PC端可操作可视化大屏，从后端拉取数据，并用flash进行渲染，实现对不同路口的信息监测。

在指挥管理端的数据监控可视化大屏-区域总览界面中，在vue框架下echars及video.js等在页面上展示了各个监测区域路口的人流量车流量相关信息，可全屏放大。通过在中央区域选择所需区域可切换相应实时监控，选择热点监控界面，可同时展示非法车辆报警监控在内的多个热门监控。

在图表栏可以鼠标点击时间栏选择所需不同时间段的数据，也可以拖动时间栏长度来改变展示的时段长度。同时使用在右上方的功能栏可以进行折线图与柱状图的转换、放大图像区域、下载当前数据图表报表和刷新数据等功能。打开人流量车流量汇总功能，选择起始时间和所需汇总区域名，可以实现人流与车流的信息汇总。

### 5.2.2网页端实现

用socket实现PC端浏览器和移动端浏览器之间的双端通信。移动端和PC端同时连接上server端的socket，两者可以同时给server端发送指令和被动接受server端主动转发过来的指令。

当移动端需要切换路口时，移动端发出切换路口指令，server端收到该指令和对应的路口，server端通过主动推送(广播)的方式转发给已经连接到server端的客户端，PC端收到来自于server端的推送后根据具体的指令和参数对页面进行重新渲染，这样就完成了一次移动端控制PC端的过程。当PC端完成更新后又会用反转的路径通知一次移动端已经完成更新，其具体过程为：PC端发送已更新成功/失败的指令给server端，server端将收到的信息主动推送(广播)给移动端，移动端收到更新成功/失败的结果后弹出成功失败的信息显示。

### 5.2.3 APP实现

我们使用h5进行手机端开发，将指令传向后端socket，拉取图表数据和video视频在前端渲染并显示。为提高操作准确度，及时发现响应，如果在前端成功完成渲染会将信息传回socket并在小屏端显示切换路口成功信息。首先打开app登录系统，而后点击界面选择所需要的区和路口，可以完成实时路口监控的切换。

有关小屏控大屏的相关功能够做到稳定跨平台、多设备操作、远程实时操作、快速便捷等方面的领先技术优势，使得用户体验更好更稳定。

### 5.3.2路口设计

本项硬件系统主要由信号灯，（Raspberry Pi 4）树莓派开发板和四个封装HDR2.54-1X4组成。

通过四个封装HDR2.54-1X4及信号灯组成十字路口交通信号灯，用来模拟路口信号灯状态。

可以通过树莓派[GPIO](https://so.csdn.net/so/search?q=GPIO&spm=1001.2101.3001.7020)输出高低电平或者通过它们读入引脚的状态（是高电平或是低电平），我们用Python来读取树莓派的CPU使用率，如果CPU负载低于50%，会显示绿灯；在50%到90%之间，显示黄灯；高于90%就会显示红灯。结合算法控制亮灭及时间长短。

同时，设计了异常处理程序，如果出现了故障，则向大屏传递故障信息，并使交通灯保持全灭。

### 5.3.3物联网模块

（1）NB-LOT物联网模块

在物联网应用中，无线通信技术起着至关重要的作用。无线通信技术按照通信距离来划分，可分为短距离通信技术和低功率广域网，即广域网通信技术（LPWAN）。LPWAN又可以分为两类：一类是在授权频谱下工作且支持GPP的蜂窝通信技术；另一类是工作于未授权频谱的Lo Ra、Sig Fox等技术。NB-LOT就属于第一类技术。其具有功耗低、覆盖广、连接多、成本低以及架构优等特点。NB-LOT采取带内、保护带或独立载波3种部署方式。与其他通信技术不同，该技术使用License频段，与现有网络可以共存。NB-LOT的中部署方式如图所示。

（2）模块优势

创新性：近几年NB-LOT成为万物物联网的一个重要发展分支之一，作为物联网通信的一个新兴技术，其最明显的特点是支持低功耗设备在广域网的蜂窝数据连接。NB-LOT模块能将数据上传到云平台进行处理。利用深度卷积神经网络辅助调节，更好地实现对人流车流的监控和调节。

针对性：该模型具有较强的针对性，模型的各环节通过无线网络将路况数据实时上传至云数据库中，用户可以随时随地查看数据信息，通过云端数据实时了解道路的车辆信息和人流信息。

稳定性：功耗一直是模型运用于实际时考虑的重点，NB-LOT技术很好地解决了目前普遍功耗较高的难题。本模型采用NB-LOT技术来传输数据，其端模块的待机时间可长达10年，解决了当前的物联网难点，为项目发展提供稳定的支持。

（3）数据采集与处理

数据收集与处理是贯穿整个模型的，由传感器收集基础数据，利用树莓派通过NB-LOT网络将数据传输到云服务台进行处理。利用BP神经网络算法进行数据处理，对数据处理后得出需要调整参数的幅度，向发送指令作出调整。

## 5.5数据安全设计

数据备份设计方案如图。

![R-C](media/image18.png){width="4.902777777777778in" height="3.745833333333333in"}

图16  数据备份设计方案图

**文件服务备份：**采用双机互为热备用技术，即一台服务器作为工作服务器，另一台作为备份服务器，并通过高速内网进行连接。当主服务器出现故障时，能自动实现故障转移，启动备份服务器，以确保服务的正常运转不中断。另外将单独设置服务器用以实时备份文件服务器。

**数据库备份：**采用MYSQL数据库服务器集群，灵活分配节点资源，当数据库主服务出现故障时，能瞬间转移故障，确保服务不中断。另外将单独设置服务器用以备份数据库文件。

**备份时间：**管理者设置自动备份时间，进行按时备份。
