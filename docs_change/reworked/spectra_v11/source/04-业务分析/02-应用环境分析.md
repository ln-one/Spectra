## 应用环境分析

| 环境类型 | 当前情况 | 说明 |
| --- | --- | --- |
| 开发者环境 | Python、TypeScript、FastAPI、React | 前后端分工明确，便于模块化开发 |
| 数据与任务环境 | PostgreSQL、Redis、RQ、Qdrant | 支持数据存储、异步任务和检索能力 |
| 用户使用环境 | Web 端工作台 | 主要面向教师在浏览器中完成操作 |
| 结果交付环境 | 支持预览、下载和标准文档导出 | 便于把生成结果直接用于备课与展示 |

从当前实现看，前后端协同环境已经比较清楚。前端主要以 Web 端工作台承接资料、会话、预览和结果操作，后端负责会话编排、检索、预览、结果绑定和任务执行。这样的环境安排符合当前作品的展示重点。

如果按使用过程来看，当前应用环境主要可以分成三层：

- 用户侧环境：教师主要通过浏览器进入工作台，在同一界面中完成资料整理、会话推进、结果查看和下载；
- 服务侧环境：后端通过接口、队列和检索组件承接资料解析、生成调度、检索调用、预览和导出相关逻辑；
- 交付侧环境：生成结果支持预览、继续修改和标准格式导出，便于直接进入备课或课堂展示场景。

当前作品不是只有页面演示。无论是资料进入、检索调用，还是预览与导出，都对应着清楚的前后端协同关系。正文在这里要说明的是：这些环节已经进入当前作品，而不是停留在概念层。
