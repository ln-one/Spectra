# PPT 插图策略方案

> 更新时间：2026-03-25
> 状态：当前生效
> 定位：成员 D 针对 PPT 插图决策的第一版策略方案

## 1. 目标

本方案解决三个具体问题：

1. 什么情况下应该插图
2. 应该插什么类型的图
3. 图片与页面文字如何形成讲解对应

它与《PPT 图片召回与素材命中优化方案》互补：

- 前者回答“素材命中是否成立”
- 本文回答“命中后是否值得插、插什么、怎么配合讲解”

## 2. 何时应该插图

优先插图的页面类型：

1. 过程讲解页
2. 结构讲解页
3. 实验现象页
4. 明显依赖观察、对比、空间关系的知识点页

默认不优先插图的页面类型：

1. 纯概念定义页
2. 结论总结页
3. 已经文字很满的页面
4. 找不到强相关素材的页面

## 3. 插什么图

按页面类型优先选择：

1. 过程讲解页：流程图、步骤图、时序图
2. 结构讲解页：结构图、分层图、标注图
3. 实验现象页：实验照片、示意图、前后对比图
4. 对比讲解页：并列对照图

默认不推荐：

1. 纯装饰背景图
2. 只和主题大致相关但不支撑本页结论的图片
3. 占空间但教师讲解无法明确指向的图片

## 4. 图文如何对应

图片进入页面后，文本必须满足：

1. 正文或讲解句明确指出“图上应该看什么”
2. 文字内容与图片对应到同一个核心结论
3. 不允许图片讲一个点，文字讲另一个点
4. 如果图片已经承担主要说明任务，文字应适当收缩，不重复描述

## 5. 当前接入方式

当前这版策略先通过两种方式接入：

1. 写入 PPT 生成 prompt 的插图决策规则
2. 作为后续人工对照和样本审计的判断口径

当前不在 D 侧直接实现真实图片插入链路；真实主链路接入仍由功能侧继续完成。
