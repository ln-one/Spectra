# 检索与重排质量改进证据

> 更新时间：2026-03-25
> 状态：当前生效
> 定位：成员 D 当前阶段关于检索与重排质量的证据汇总

## 1. 结论先行

当前主线已经具备一组可以复现的检索与排序质量证据，但证据强度分层如下：

1. `检索命中与稳定性`：证据较强，已有独立 baseline
2. `来源可读性与相关性`：证据中等，已有抽样 baseline
3. `网络资源排序/筛选质量`：证据较强，已有 gate 与 baseline
4. `主链路内部更细粒度 reranking`：仍应谨慎表述，当前不宜过度声称“所有重排问题都已解决”

因此，这份文档的目标不是夸大，而是明确：

1. 现在能证明什么
2. 现在还不能证明什么
3. 后续优化建议应基于哪些已有证据

## 2. 检索命中与稳定性证据

证据来源：

1. `backend/eval/run_eval.py`
2. `backend/eval/baseline_manager.py`
3. `backend/eval/baselines/rag-baseline-v1.json`

当前已冻结的真实项目基线事实：

1. `keyword_hit_rate = 0.9`
2. `failure_rate = 0.0`
3. `avg_latency_ms = 743.7`
4. `failed_case_ids = []`

这意味着：

1. 当前检索链路在真实项目样本上已经能稳定命中大多数目标关键词
2. 当前基线没有出现评测级失败样本
3. 检索时延已经落在可接受的可观测范围内

当前能成立的说法是：

1. 检索链路已具备“可复现 baseline + 可回归校验”的质量基础
2. 当前阶段不是完全没有检索能力，而是需要继续提高命中解释力和展示价值

## 3. 来源可读性与相关性证据

证据来源：

1. `backend/eval/source_quality_audit.py`
2. `backend/eval/source_quality_baseline.py`
3. `backend/eval/baselines/source-quality-baseline-v1.json`

当前 baseline 事实：

1. `coverage_rate = 1.0`
2. `readability_rate = 1.0`
3. `relevance_rate = 0.5`
4. `failed_sample_ids = ["audit-002"]`

这说明：

1. 当前来源字段覆盖率和可读性已经较好
2. 但“来源是否真正和回答高度相关”还不是满分
3. 因此当前更适合说“来源展示已具备可用基础”，而不是说“来源相关性已经完全收口”

对检索/重排工作的意义是：

1. 检索不是只看命中，还要看命中的内容是否真的支撑回答
2. 后续质量优化应继续围绕 `relevance_rate` 提升，而不是只追求返回更多来源

## 4. 网络资源排序与筛选证据

证据来源：

1. `backend/eval/network_resource_quality_audit.py`
2. `backend/eval/network_resource_baseline.py`
3. `backend/eval/baselines/network-resource-baseline-v1.json`

当前 baseline 事实：

1. `normalization_rate = 1.0`
2. `relevance_pass_rate = 1.0`
3. `low_quality_reject_rate = 1.0`
4. `citation_ready_rate = 1.0`
5. `gate_passed = true`

这组结果能支持的结论是：

1. 当前网络资源策略层已经能把输入资源标准化为可入库知识单元
2. 排序后的高位结果与查询目标保持相关
3. 低质量或弱相关资源能够被过滤
4. 输出结果已经具备后续引用和入库准备度

这是当前“重排质量”最强的一组证据。

## 5. 路由与质量收益的旁证

证据来源：

1. `backend/eval/router_quality_audit.py`
2. `backend/eval/router_quality_baseline.py`
3. `backend/eval/baselines/router-quality-baseline-v1.json`

当前 baseline 事实：

1. `quality_delta = -0.0086`
2. `latency_reduction_rate = 0.2281`
3. `cost_reduction_rate = 0.2153`
4. `non_degradable_misroute_rate = 0.0`
5. `gate_passed = true`

这不是直接的检索重排证据，但能说明：

1. 质量优化不是单点脚本，而是和主链路路由策略共同作用
2. 当前系统已经在“质量不显著退化”的前提下，获得延迟与成本收益

## 6. 当前能正式对外表述的内容

当前阶段可以稳妥对外表述为：

1. Spectra 已具备可复现的检索 baseline，真实项目样本上 `keyword_hit_rate` 达到 `0.9`
2. 来源字段覆盖率和可读性已达到 `1.0`
3. 网络资源排序/筛选链路已通过独立质量门禁
4. 检索、来源、资源排序质量都已进入“有 baseline、有 guardrail、可回归”的阶段

## 7. 当前不应过度声称的内容

当前不应直接声称：

1. 所有主链路内部 reranking 都已经完全优化完成
2. 来源相关性已经达到强一致
3. 检索命中已经足以覆盖所有教学场景

原因很明确：

1. `source-quality-baseline` 中 `relevance_rate` 仍为 `0.5`
2. 当前更强的重排证据主要来自网络资源策略层
3. 仍需继续用真实样本补足“检索结果为什么更适合教学”的专项证明

## 8. 当前阶段的直接使用方式

这份证据当前可直接服务于：

1. `PPT / 卡片 / 大纲` 质量优化的资料利用说明
2. `初审材料` 中“检索与引用不是黑箱”的技术亮点表达
3. 后续主线接入建议中的质量依据

## 9. 后续建议

后续如果继续做检索与重排优化，默认优先补这三类证据：

1. 提升 `source relevance` 的专项样本
2. 补充“为什么这个结果更适合教学场景”的排序案例
3. 继续把网络资源排序层的证据和主链路 RAG 命中证据接起来
