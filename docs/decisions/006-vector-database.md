# ADR-006: 向量数据库选择 - ChromaDB

**状态**: 已接受 
**日期**: 2026-02-19

## 背景

需要向量数据库存储教材的 Embedding 分段，支持语义检索和 RAG 增强生成。

## 考虑的方案

| 方案 | 优势 | 劣势 |
|------|------|------|
| **ChromaDB** | 极轻量，单机运行，开发成本极低 | 高并发性能一般 |
| Milvus | 分布式架构，适合大规模数据 | 部署非常重，需多组件 |
| Pinecone | 云托管，无需运维 | 收费，数据在国外 |
| Qdrant | 性能好，支持混合检索 | 社区生态不如 ChromaDB |

## 决策

初期使用 **ChromaDB**，后期可迁移至 Milvus

## 理由

### 开发效率
- 几行代码即可完成集成
- 零配置启动，单文件存储
- Python 原生支持

### 性能满足
- 检索延迟约 100ms
- 满足教学场景的实时性要求
- 支持元数据过滤

### 迭代友好
- 适合快速原型和 Demo
- 后期数据迁移成本可控

## 权衡

- 不支持分布式 → 初期单机够用
- 高并发受限 → 教育场景并发不高
- 迁移 Milvus 需要工作 → 预留窗口

## 影响

- 后端集成 ChromaDB
- 知识库入库时生成 Embedding
- 检索时结合 BM25 + 向量搜索
