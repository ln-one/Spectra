# 5-3 Session 生成与 refine 闭环图

## 版本

`答辩版`

## 适配场景

`PPT 横向`

## 图类型

`闭环 / 主链图`

## 这张图只回答什么

为什么 `Session` 内的生成与 refine 不是单次调用，而是带确认、预览、修改的可控闭环。

## 主阅读路径

从左到右看生成主链，再看中段的 refine 反馈。

## 来源与事实锚点

- `docs/competition/05-key-technologies.md`
- `docs/competition/05-key-technologies-src/01-session-loop.md`
- session generation 相关实现

## 现有图问题检测

- 容易画成无反馈单向链
- 容易把 preview 当成最终 owner
- `结论`：`需中度重构`

## 信息分层设计

- 输入与上下文
- 大纲与生成
- 预览与 refine
- 导出结果

## 分组设计

- 左：上下文与证据
- 中：生成控制点
- 右：预览导出

## 密度策略

- `中密度`
- 保留关键 refine 控制点，不做过度简化

## 画幅与布局约束

- `16:9` 横向
- 三段主链
- 允许一条局部反馈

## 优化后的 Mermaid 骨架

```mermaid
flowchart LR
    A["Session Context"] --> B["大纲"]
    B --> C["确认"]
    C --> D["生成"]
    D --> E["预览 / 修改"]
    E --> F["导出"]
    E --> D
```

这张图是 `16:9` 横向图。  
主链必须是 `Session Context -> 大纲 -> 确认 -> 生成 -> 预览/修改 -> 导出`，并保留 `预览/修改 -> 生成` 的局部反馈。  
画面要体现“可控生成”，而不是一键出结果。
