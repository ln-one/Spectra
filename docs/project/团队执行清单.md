# 团队执行清单

> 更新时间：2026-03-24
> 状态：当前生效
> 定位：当前阶段已明确任务的执行清单

本文档只放执行项，不放职责解释。

使用规则：

- 未完成的事项必须保持 `- [ ]`
- 只有在真正完成、已验证、可演示后，才能改成 `- [x]`
- 做完必须立即打勾，不允许口头完成、文档不更新
- 如果事项过大，必须先拆成更细子事项，再继续推进

特别说明：

- 本清单覆盖当前已经明确的核心需求与任务
- 但本清单不是全部工作清单
- 清单之外，仍然会存在很多在联调、跑系统、准备演示、修主链路时暴露出来的新任务
- 因此，不能把“清单里的都打勾了”等同于“项目已经全部做完”
- 正确理解应该是：清单里的事项是必须完成的明确任务，但不是完整上限
