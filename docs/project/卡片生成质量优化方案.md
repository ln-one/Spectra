# 卡片生成质量优化方案

> 更新时间：2026-03-25
> 状态：当前生效
> 定位：成员 D 针对 Studio 卡片结果质量的第一版优化方案

## 1. 目标

当前阶段不把“卡片功能是否存在”作为 D 的主目标，而把重点放在：

1. 结果是否可演示
2. 内容是否贴合教学场景
3. 是否有效利用资料、检索和提示词

## 2. 优先优化对象

当前优先关注以下卡片：

1. `interactive_quick_quiz`
2. `knowledge_mindmap`
3. `demonstration_animations`
4. `speaker_notes`
5. `classroom_qa_simulator`

原因：

1. 这几类卡片更容易直接展示质量差异
2. 也最能体现“检索 + 提示词 + 教学表达”的综合效果

当前暂不把以下对象作为第一优先级：

1. `word_document`
2. `interactive_games`

原因：

1. `word_document` 更接近主线文档生成能力，当前 D 更适合先做结果质量方法，而不是把文档链路当作卡片专项主目标
2. `interactive_games` 更偏功能丰富度和前端演示复杂度，当前阶段更适合先优化更容易形成“质量前后对照”的卡片类型

## 3. 质量判断维度

卡片质量默认从四个维度判断：

1. 教学相关性
   - 是否服务当前主题和课堂目标
2. 结构可用性
   - 是否具备清晰结构，而不是结果碎片
3. 资料利用度
   - 是否真实吸收已有资料，而不是空泛生成
4. 演示可用性
   - 是否能被截图、录屏、课堂讲解直接使用

## 4. 默认优化路径

卡片质量默认按以下顺序优化：

1. 先做样本归类
2. 再补提示词规则
3. 再看资料与素材利用是否不足
4. 最后再决定是否需要主链路侧补功能

这样可以把 D 的质量工作和 C 的功能工作拆开，减少反复。

## 5. 当前输出要求

后续如果要证明卡片质量提升，默认至少给出：

1. 一组问题样本
2. 一组优化后样本
3. 对应提示词或规则变化
4. 对课堂使用价值的解释
