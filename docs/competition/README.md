# Competition Docs

## 定位

本目录存放 `Spectra` 服务外包大赛提交材料的 Markdown 主稿、附录与内部维护文件。

当前主叙事已经统一为：

`库是系统本体 -> 课程数据库是长期沉淀对象 -> 多模态内容是按需外化结果 -> 引用关系形成课程知识网络`

## 维护清单

- 正式提交正文使用 `00-executive-summary.md` 到 `08-requirements-coverage.md`。
- 正式提交正文必须保持“库 / 版本 / 导出物”语义边界清晰，避免回退到“只是在分享 PPT 文件”的表述。
- `00-writing-guide.md` 和 `99-self-review.md` 仅供内部维护，不进入正式提交正文。
- `11-final-submission-draft.md` 是合并审阅稿，源章节更新后应重新生成，不应手工长期维护两套正文。
- 调整章节顺序、文件名或删除文档时，必须同步修复本目录链接与主稿入口。
