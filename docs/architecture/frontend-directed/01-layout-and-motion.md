# 前端导向设计：布局与动效

## 1. 项目页核心布局（`/projects/[id]/`）

采用三栏式响应布局：
- 左栏 `Studio`：工具与模式切换。
- 中栏 `Conversation`：对话与生成交互主舞台。
- 右栏 `Sources`：素材与 RAG 资源管理。

关键模块：
- `Studio Toolbelt Card`：工具/历史切换，模式激活与推荐素材联动。
- `Intelligent Chat Hub`：多轮对话、引用状态、反馈动作。
- `Multimodal Asset Library`：上传、检索、勾选素材范围。
- `Global Header`：跨项目入口、全局通知与系统级操作。

## 2. 动效与空间流转

动效原则：
- 以弹簧缓动为主，强调反馈与状态可感知。
- 动效服务于状态表达，不应遮蔽业务信息。

关键转场（聊天模式 -> 生成工作台）：
- `Studio Expand`：左栏扩展为主工作区。
- `Conversation Shrink`：中栏收缩为辅助浮窗。
- `Sources Collapse`：右栏折叠为可快速展开的图钉态。

工程约束：
- 低性能设备自动降级为淡入淡出，不阻塞交互。
- 所有动效必须可中断（状态跳转时立刻收敛到目标态）。
- 状态优先于动画：当两者冲突时以状态渲染正确性为准。
