# 最终模型路由矩阵

以下为当前项目的最终模型路由配置。  
整体原则是：在**不改动业务逻辑**的前提下，仅针对不同任务选择更合适的模型；所有模型均通过 **DashScope** 统一调用。  
从当前结果来看，现有配置已经完成优化，**暂时无需进一步调整**。

## 配置说明

### 1. 轻量任务
以下任务以低延迟、低成本和较快响应为优先目标，因此统一选择 `qwen3.5-flash`：
- 意图理解
- 标题优化
- 短文本优化
- 流式对话-无 RAG

### 2. 中高复杂度生成与整合任务
以下任务对内容理解、信息整合、表达质量或长文本生成能力要求更高，因此统一选择 `qwen3.5-plus`：
- 流式对话-有 RAG
- 大纲生成
- PPT 正文生成
- 课件修改
- RAG 深度摘要
- Style Fallback（JSON 输出）

### 3. 特殊生成任务
在 `LESSON_PLAN_REASONING` 相关任务中：
- `Render Rewrite` 选择 `MiniMax-M2.5`（完整 Marp 文档重写，需要长文生成能力）
- `Style Fallback` 选择 `qwen3.5-plus`（JSON 输出，已修复）

这说明当前课件生成链路中，不同子阶段已经根据生成目标做了进一步细分。

## 修复记录

### 2026-04-06：修复 Style Fallback 模型选择
- **问题**：Style Fallback 错误地使用了 `MiniMax-M2.5`，但该任务只需输出 JSON 格式的样式契约
- **修复**：移除显式 `model=RENDER_REWRITE_MODEL` 参数，让其走默认路由（`qwen3.5-plus`）
- **影响**：Render Rewrite 仍使用 MiniMax-M2.5，Style Fallback 改用 qwen3.5-plus

## 结论

当前模型配置已优化完成，整体分配逻辑清晰：
- 轻量快速任务使用 `qwen3.5-flash`
- 需要更强生成与整合能力的任务使用 `qwen3.5-plus`
- 个别特殊生成环节使用 `MiniMax-M2.5`
