### 📄 1. Word 教案与文档 (Document Generator)

- **核心作用**：  
  将教学大纲和 RAG 知识库转化为标准化、可打印的教学文档，并原生支持“分层教学”理念。

- **交互方式**：
  
  - **卡片展开与配置**：在 Studio 面板点击该卡片后，卡片平滑放大撑满屏幕左侧至中间（约占 75% 宽度）。卡片内部首先渲染配置区：用户选择需要生成的类型（如：分层教案、学生讲义、课后测试题、实验指导书等）。选中“分层教案”后，卡片内手风琴式向下展开深层选项（如：教学模型、适用年级）。
  
  - **内容呈现**：点击生成后，配置项自动收起或滑出，卡片内部直接渲染为 Markdown/富文本阅读器。
  
  - **上下文感知微调**：用户视线转向右上角悬浮的 **Chat 面板**。由于 Chat 已识别当前处于“Word 文档”上下文中，用户直接输入：“把 B 层讲义里的第二道例题换成更生活化的场景”，Studio 卡片内的文档视图立即产生高亮重绘动画，完成局部替换。

### 📝 2. 随堂小测 (Interactive Quick Quiz)

- **核心作用**：  
  自动生成贴合当前知识点的测验题，摒弃传统静态试卷，采用沉浸式的“单题交互”模式进行效果检验。

- **交互方式**：
  
  - **卡片展开与配置**：卡片展开后，内部显示题型设置（如题量、难度、是否加入幽默干扰项），确认后生成。
  
  - **内容呈现**：展开的卡片内部转变为“闪卡（Flashcard）”沉浸视图。屏幕中央只显示一道题及选项。点击错误选项时，该选项下方平滑展开详细解析（分析错误逻辑），阅读后点击卡片右下角的 [下一题 ➔]。
  
  - **上下文感知微调**：在右上角 **Chat 面板**输入：“这道题太简单了，帮我换一道考察易错点的多选题”。此时，Studio 展开卡片内的当前题目瞬间翻转、重绘为新题目。

### 🎮 3. 互动游戏 (AI-Driven Interactive Games)

- **核心作用**：  
  发挥大模型的代码生成能力，根据课程内容“无中生有”地创造 HTML/JS 情景互动游戏（如连线、排序、解谜）。

- **交互方式**：
  
  - **卡片展开与配置**：卡片展开后，呈现一个极简的输入框或几个灵感 Tag（如：“时间轴排序”、“概念连线”）。用户可以选择让 AI 自由发挥。
  
  - **内容呈现**：AI 处理完毕后，展开的卡片内部直接嵌入一个可实际点击游玩的 **Iframe/沙盒（Sandbox）**，渲染游戏画面。
  
  - **上下文感知微调**：在右上角 **Chat 面板**输入：“给这个连线游戏加一个 30 秒的倒计时限制，超时算失败”，AI 后台重写代码，Studio 卡片内的游戏沙盒自动热更新（Hot Reload）。

### 🗺️ 4. 思维导图 (Knowledge Mind Map)

- **核心作用**：  
  提取复杂的多模态素材逻辑，生成高度结构化的树状知识图谱，帮助学生建立全局观。

- **交互方式**：
  
  - **内容呈现**：点击卡片展开后，系统自动基于当前的 RAG 知识库生成导图，卡片内部利用 Markmap 实时渲染 SVG 视图。支持在卡片内用鼠标拖拽漫游（Pan）和缩放（Zoom）。
  
  - **上下文感知微调**：用户在展开的卡片中点击选中某个特定节点（如“细胞核”），然后在右上角 **Chat 面板**输入：“把这个节点向下再拆分两级”。由于 Chat 监听了卡片的选中状态，它会精准控制导图在卡片内平滑生出新的子分支。

### 🎬 5. 演示动画 (Demonstration Animations)

- **核心作用**：  
  把物理、计算机等学科中难以用文字描述的抽象动态过程（如：冒泡排序、磁感线变化），转化为可视化代码动画。

- **交互方式**：
  
  - **内容呈现**：卡片展开后，内部分为“代码区（可选折叠）”和“渲染区”。直接播放大模型生成的 Echarts 动态配置项或前端 Canvas 绘制的动画。
  
  - **上下文感知微调**：在右上角 **Chat 面板**直接输入：“把动画里粒子的运动速度调慢一半，并把轨迹改成红色虚线”。Studio 卡片内的动画视图实时接收新参数并重载生效。

### 🎙️ 6. 说课助手 (Speaker Notes / Pitch Assistant)

- **核心作用**：  
  为老师提供“手把手的舞台剧本”，包含破冰开场白、页面过渡语、时间分配及肢体动作提示。

- **交互方式**：
  
  - **卡片展开与配置**：卡片展开后，内部提示“请选择已生成的 PPT 项目”。选择后，系统开始生成剧本。
  
  - **内容呈现**：卡片内部切换为“提词器视图”。左侧是 PPT 缩略图滚动列表，右侧是超大字号的演说稿文本，其中穿插着紫色的动作提示（如：[动作：在此处停顿3秒，环顾教室]）。
  
  - **上下文感知微调**：高亮选中卡片内的一段过渡语，在右上角 **Chat 面板**输入：“这段引入太生硬，用一个关于‘黑神话悟空’的比喻来改写”，卡片内对应段落产生高亮替换动画。

### 🤔 7. 学情预演 (Classroom Q&A Simulator)

- **核心作用**：  
  课前沙盘推演。让大模型扮演不同性格的“虚拟学生”对教案发起提问，帮助教师提前准备应对策略。

- **交互方式**：
  
  - **内容呈现**：卡片展开后，内部 UI 变身为“虚拟教室群聊”。几位带有不同性格 Tag 的虚拟学生（如 [发散思维的好学生], [容易搞混概念的差生]）的头像相继弹起，并抛出针对当前课件的“刁钻问题”。教师可以直接在卡片内的输入框“练手”回答，系统会给出评价和改进建议。
  
  - **上下文感知微调**：如果教师觉得当前预演的难度不够，可在右上角的**全局 Chat 面板**对“总控 AI”下发指令：“让学生多问一些关于底层公式推导的细节问题”。Studio 卡片内的虚拟学生立刻改变提问画风，发起新的提问攻势。
