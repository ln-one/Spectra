# 5 系统设计

## 5.1 系统整体架构

### 5.1.1 LLM 结合知识库的整体架构

Spectra 采用 "分层微服务 + 统一知识中枢" 的整体架构，以课程资料库（Course Knowledge Hub）为核心枢纽，向上支撑教师备课生成、学生访问学习、教研协作管理三大核心业务，向下整合多模态解析、RAG 检索增强、大模型智能编排、标准化文档生成等基础能力。架构整体遵循高内聚低耦合原则，各模块独立部署、可插拔扩展，同时通过统一的库编排引擎实现所有内容的一致性管理，确保不同模态、不同终端的内容同源同步，从根本上解决传统工具内容碎片化、版本混乱、复用困难的问题。

本架构区别于普通 AI 生成工具的核心特征在于：所有内容的生成、修改、沉淀与分发都围绕统一的课程资料库展开，PPT、教案、思维导图、动画等都只是库在不同场景下的外化导出物，而非独立的核心对象。这种设计实现了 "一次生成、多处复用、一处更新、全网同步" 的能力，为无感资产沉淀和师生双边服务提供了底层支撑。

本项目构建了 "多模态解析入库 统一知识存储 RAG 增强检索 LLM 智能编排 库化沉淀输出" 的闭环技术架构，将大模型能力与课程知识库深度融合，既发挥了大模型的理解与生成优势，又通过知识库保障了内容的准确性、一致性和可追溯性。架构自下而上分为六层：

1\. 多模态数据接入层

作为系统的统一数据入口，负责接收并标准化处理所有类型的教学输入，包括教师的自然语言指令和各类格式的教学资料。

- 文本指令处理：支持文字、语音两种输入方式，语音通过 Whisper 模型转写为文本后进入后续流程

- 文档解析：通过 PyPDF2、pythondocx、pythonpptx 等工具解析 PDF、Word、PPT 文件，精准提取标题层级、段落文本、公式、图表、注释等结构化内容

- 多媒体解析：通过 PaddleOCR 识别图片中的文字与公式，通过 FFmpeg 抽取视频关键帧并结合 Whisper 提取字幕与语音内容，生成视频片段语义摘要

- 输入预处理：对所有解析后的内容进行清洗、去重、格式标准化，为后续入库和检索做准备

2\. 统一知识存储层（核心中枢）

即课程资料库，是整个架构的核心，分为向量知识库和结构化知识库两部分，所有内容先入库再参与生成，而非临时处理。

- 向量知识库：采用 ChromaDB 轻量级向量数据库，存储分块后的文本向量、图片特征向量、视频片段语义向量，支持高效的语义检索

- 结构化知识库：采用 MySQL 关系型数据库，存储课程结构、知识点关系、版本信息、权限配置、引用关系、用户数据等结构化信息

- 库元数据管理：维护每个课程库的创建信息、更新记录、访问权限、引用依赖关系，支持版本回溯和冲突检测

3\. RAG 检索增强层

负责根据当前任务需求，从统一知识存储层中精准召回相关内容，为大模型生成提供事实依据，降低幻觉风险。

- 混合检索机制：结合语义检索和关键词检索，同时支持按学科、学段、章节、版本等元数据进行过滤，提高检索准确率

- 引用关系检索：自动召回当前课程库绑定的主基底库和辅助引用库中的相关内容，实现跨库知识复用

- 结果重排序：采用交叉编码器对初步检索结果进行重排序，筛选出最相关的 TopK 内容

- 上下文拼接：将重排序后的内容按照逻辑关系拼接成符合大模型输入要求的上下文，同时标注内容来源

4\. LLM 智能编排层

大模型的调度与管理中心，采用多模型兼容架构，支持接入通义千问、ChatGLM、Llama 等多个主流大模型，根据任务复杂度自动选择最优模型。

- 教学意图理解：解析教师的自然语言指令，提取教学目标、重点难点、讲授逻辑、互动要求等核心信息

- 多轮对话追问：识别模糊需求并主动引导教师补充信息，最终形成结构化的教学需求文档

- 内容生成控制：根据教学需求和检索上下文，规划课程内容结构，生成 PPT、教案、互动内容等多模态成果

- 反馈处理：解析教师的修改意见，定位需要调整的内容模块，驱动局部再生成

- 内容校验：对生成内容进行初步的事实校验和合规性检查，过滤错误和敏感内容

5\. 业务逻辑与库编排层

负责核心业务逻辑的实现，其中库编排引擎是本层的核心，承担着连接大模型生成与知识沉淀的关键作用。

- 用户与权限管理：实现基于角色的权限控制，区分教师、学生、管理员、协作者等不同身份的访问权限

- 版本管理：维护课程库的正式版本记录，支持版本对比、回滚和上游更新同步

- 引用与协作管理：处理库之间的引用关系，支持多人协作备课和候选变更审核

- 库编排引擎：将大模型生成的内容结构化后写入课程资料库，维护知识节点之间的逻辑关系，确保所有导出内容都来自统一的库源

- 资产沉淀：自动捕获备课过程中的所有内容和修改记录，实现无感化资产沉淀

6\. 前端应用与导出层

面向用户的交互界面和标准化输出模块，提供简洁易用的操作体验和多格式导出能力。

- 教师工作台：集成需求输入、资料上传、预览修改、库管理、分享等功能

- 学生学习空间：提供课程内容查看、个性化学习材料生成、个人知识管理等功能

- 管理员后台：支持用户管理、系统配置、数据统计、安全审计等功能

- 标准化导出引擎：采用 Marp CLI 生成 PPTX 和 HTML5 课件，采用 Pandoc 生成 DOCX 教案，同时支持 PDF、思维导图、GIF 动画等多种格式导出，所有导出物均标注对应的库版本号

### 5.1.2 实现流程

一、教师端备课生成与迭代优化流程（主流程）

1.  教师在工作台输入教学主题、学科、学段等基础信息，或上传已有的 PDF、Word、PPT 等教学资料

2.  LLM 智能编排层解析输入内容，识别模糊需求并主动追问补充，最终生成结构化的教学意图文档

3.  多模态数据接入层解析所有上传资料，提取结构化内容和知识特征，写入当前课程库的临时工作区

4.  RAG 检索增强层根据教学意图，从临时工作区、教师个人历史库、学校校本库、公共权威知识库中召回相关内容，经过重排序后形成生成上下文

5.  LLM 基于教学意图和检索上下文，先生成课程内容大纲，提交教师预览确认

6.  教师确认大纲后，LLM 分模块生成 PPT 课件和 Word 教案内容，库编排引擎同步将生成的内容结构化写入课程库临时工作区

7.  教师在预览界面查看生成结果，提出局部修改意见（如增加案例、调整结构、简化文案）

8.  系统根据修改意见定位对应内容模块，重新检索相关内容并完成局部再生成，更新课程库中的对应节点

9.  教师确认最终内容后，库编排引擎将临时工作区的内容合并为正式库版本，生成唯一版本号，完成资产沉淀

10. 教师可选择导出各类格式的文件，或直接分享课程库链接给学生

二、课程资料库无感沉淀与更新流程

1.  从教师输入需求开始，库编排引擎会自动捕获所有教学逻辑、知识结构、素材关系和修改记录，实时写入课程库临时工作区

2.  教师的每一次保存、预览、生成操作，系统都会自动同步对应内容到库中，无需任何额外的上传或整理步骤

3.  当教师发布正式版本时，系统自动生成版本快照，记录版本号、更新时间、更新内容和更新人，保留完整的历史版本记录

4.  教师后续对课程内容的任何修改，都会自动生成新的版本，支持版本对比和一键回滚

5.  课程库内容更新后，系统会自动向所有被授权的学生发送更新通知，学生访问时将直接获取最新版本内容

6.  对于基于父库派生的子库，系统会实时检测上游父库的更新，提示下游用户是否同步更新，避免内容不一致

三、学生端访问与个性化再生成流程

1.  教师生成课程库分享链接或邀请码，设置访问权限和有效期，发送给学生

2.  学生通过链接或邀请码注册登录后，进入对应的课程空间，可查看教师发布的所有正式版本内容

3.  学生可直接下载 PPT、教案、视频等各类格式的课程文件，或在线查看动态更新的内容

4.  学生输入个性化学习需求（如生成思维导图、复习提纲、知识点摘要、章节练习题）

5.  系统从对应的教师课程库中召回相关内容，经过 RAG 增强后，由 LLM 生成个性化学习材料

6.  学生生成的内容自动沉淀到个人学习空间，与教师的权威课程库完全隔离，不会污染教师侧内容

7.  学生可在个人学习空间中整理、编辑和管理自己的学习资料，支持引用教师课程库中的内容构建个人知识体系

# 5.2 软件技术

本项目采用工业级成熟开源技术栈构建，严格遵循前后端分离、分层解耦、可插拔扩展的设计原则，所有技术选型均围绕"可实现、可交付、可维护、可扩展"四大核心目标，兼顾开发效率、运行性能与部署成本，完全适配教育服务外包项目的交付要求与长期运营需求。

### 5.2.1 前端框架与状态管理（Next.js + TypeScript + Zustand）

前端采用Next.js 14 App Router + TypeScript 5.4 + Zustand 4.5的黄金组合，打造高性能、高可用、体验流畅的Web工作台。

技术选型依据与核心优势

- Next.js 14：采用服务端渲染（SSR）与客户端渲染（CSR）混合模式，首屏加载速度比纯React单页应用提升40%以上；内置路由系统、图片优化、API代理与静态导出能力，无需额外配置即可实现多终端自适应；支持Edge Runtime部署，降低服务器成本的同时提升全球访问速度。

- TypeScript：全链路强类型约束，在开发阶段提前捕获90%以上的类型错误，大幅减少线上bug；提供完善的代码提示与重构支持，显著提升团队协作效率与代码可维护性，适合长期迭代的大型项目。

- Zustand：替代传统Redux架构的轻量级状态管理方案，无样板代码、学习成本低、打包体积小；支持全局状态分片、中间件扩展与持久化存储，完美适配本项目中用户状态、生成任务进度、课程库数据等多维度状态管理需求。

项目具体应用

- 教师工作台采用客户端渲染，保证交互响应速度；学生端与分享页面采用服务端渲染，提升加载速度与SEO友好度

- 基于ShadCN UI构建统一的设计系统，实现按钮、表单、弹窗、预览等组件的标准化复用

- 使用React Hook Form处理复杂表单校验，Zod进行前后端统一的数据结构校验

- 集成Marp预览组件，实现PPT课件的实时在线预览与编辑

### 5.2.2 后端框架与分层服务（FastAPI + Router/Service）

后端采用FastAPI 0.110 + Python 3.11构建高性能RESTful API服务，严格按照路由层服务层数据访问层三层架构实现业务解耦。

1.技术选型依据与核心优势

- FastAPI：基于Python的异步Web框架，性能接近Node.js与Go语言，是当前Python生态中性能最高的Web框架；支持自动生成Swagger/OpenAPI接口文档，无需手动编写；内置依赖注入系统，便于实现权限验证、日志记录等横切关注点；原生支持异步请求处理，非常适合大模型调用、文件解析等IO密集型任务。

- 分层架构：将业务逻辑与路由、数据访问严格分离，每个模块职责单一，便于单元测试与独立部署；服务层封装核心业务逻辑，可被多个路由复用，避免代码冗余。

项目具体应用

- 路由层：负责接收HTTP请求、参数校验、权限验证与请求转发，不包含任何业务逻辑

- 服务层：封装课程生成、知识库管理、文件解析、权限控制、任务调度等核心业务功能

- 数据访问层：通过ORM与数据库交互，保证数据操作的原子性与一致性

- 依赖注入：实现用户认证、日志记录、异常处理等通用功能的统一管理

- 中间件：添加CORS跨域、请求限流、访问日志、异常捕获等全局中间件

### 5.2.3 数据存储与对象建模（PostgreSQL + Prisma）

结构化数据采用PostgreSQL 16 + Prisma 5.12实现稳定、安全、高效的数据持久化管理。

技术选型依据与核心优势

- PostgreSQL：企业级开源关系型数据库，支持复杂查询、JSONB字段、全文检索、事务安全与高并发读写；具备完善的数据完整性约束与安全机制，满足教育行业对数据安全与合规的严格要求；支持水平扩展与读写分离，能够支撑百万级用户规模。

- Prisma：下一代类型安全ORM框架，通过Schema定义数据模型，自动生成类型安全的数据库访问客户端；支持自动数据库迁移、联表查询、事务管理与性能监控；与TypeScript无缝集成，提供完善的代码提示与类型检查，大幅降低数据库操作的出错概率。

项目核心数据模型

- 用户域：用户表、角色表、权限表、组织表

- 课程域：课程库表、版本表、知识点表、引用关系表、协作者表

- 资源域：文件表、文档片段表、向量索引表、任务记录表

- 系统域：配置表、日志表、通知表、审计表

项目具体应用

- 使用PostgreSQL的JSONB字段存储课程结构、配置信息等半结构化数据

- 利用PostgreSQL的全文检索能力实现课程库与资源的快速搜索

- 通过Prisma的事务机制保证课程生成、版本更新等关键操作的原子性

- 建立完善的索引体系，优化高频查询的性能

### 5.2.4 向量检索与检索库（Qdrant + Stratumind）

RAG检索增强系统采用Qdrant 1.9 + Stratumind 0.3构建高精度、高性能的向量检索体系。

技术选型依据与核心优势

- Qdrant：开源高性能向量数据库，专门为大规模向量相似度检索设计；支持毫秒级查询、丰富的过滤条件、动态索引更新与分布式部署；提供完善的RESTful API与Python客户端，易于集成与维护；相比ChromaDB、Milvus等其他向量数据库，Qdrant在检索精度、查询速度与部署便捷性之间达到了最佳平衡。

- Stratumind：专为教育场景优化的检索增强框架，提供智能文档分块、多模态向量化、混合检索、结果重排序等全链路能力；内置教育领域专用的分块策略与向量化模型，能够显著提升教学资料的检索准确率。

项目具体应用

- 文档分块：采用语义分块策略，将教学资料按知识点、章节进行智能分块，保留完整的语义上下文

- 向量化：使用bgelargezhv1.5模型生成中文向量，兼顾检索精度与推理速度

- 混合检索：结合语义检索与关键词检索，同时支持按学科、学段、章节、版本等元数据过滤

- 结果重排序：使用交叉编码器对初步检索结果进行重排序，筛选出最相关的Top10内容

- 增量更新：支持文档的增量更新与删除，保证向量库与原始文档的一致性

### 5.2.5 异步任务与队列执行（Redis + RQ + Worker）

针对课件生成、文档解析、大模型调用等耗时任务，系统采用Redis 7.2 + RQ 1.16 + 独立Worker进程实现异步任务队列管理。

技术选型依据与核心优势

- Redis：高性能内存数据库，用作任务队列存储、结果缓存、接口限流与状态共享；支持多种数据结构，具备极高的读写性能与可靠性；部署简单、运维成本低，非常适合中小型项目的异步任务场景。

- RQ（Redis Queue）：轻量级Python异步任务队列，基于Redis实现，简单稳定、易于部署与调试；支持任务优先级、失败重试、超时控制与进度追踪；相比Celery等更复杂的任务队列，RQ的学习成本更低、维护更简单，完全满足本项目的需求。

项目具体应用

- 异步任务类型：文档解析、向量入库、课件生成、教案生成、文件导出、批量处理

- 任务队列分级：分为高优先级（实时生成任务）、中优先级（批量处理任务）、低优先级（后台统计任务）

- Worker部署：独立部署多个Worker进程，分别处理不同优先级的任务，支持动态扩缩容

- 任务监控：集成RQ Dashboard实现任务的可视化监控与管理，支持失败任务的手动重试

- 进度追踪：实时更新任务进度，前端通过轮询获取任务状态

### 5.2.6 文档上传与解析能力（Dualweave + MinerU）

多模态教学资料解析采用Dualweave 2.1 + MinerU 0.7实现全格式、高精度的文档解析。

技术选型依据与核心优势

- MinerU：开源高性能文档解析工具，专门针对中文教育文档优化；支持PDF、Word、PPT、Excel等多种格式的解析，能够精准提取标题层级、段落文本、公式、图表、表格、注释等内容；保持原始文档的排版结构与逻辑关系，解析准确率远高于PyPDF2、pythondocx等传统工具。

- Dualweave：增强型文档处理引擎，提供扫描件OCR识别、手写体识别、表格结构还原、页眉页脚清洗、水印去除等高级功能；支持大文件分片上传与断点续传，能够处理GB级别的大型教学资料。

项目具体应用

- 支持的文件格式：PDF（原生/扫描件）、DOCX、PPTX、XLSX、JPG、PNG、MP4、MP3

- 解析流程：文件上传 → 格式识别 → 内容提取 → 结构还原 → 语义分块 → 向量入库

- 公式处理：支持LaTeX公式与图片公式的识别与转换，生成可编辑的LaTe**X**代码

- 表格处理：精准还原表格结构，支持复杂合并单元格的解析

- 进度反馈：实时返回解析进度与状态，解析失败时提供详细的错误信息

### 5.2.7 多模型路由与推理编排（Qwen/Minimax Router）

本项目构建模型无关的多模型路由与推理编排层，避免单一大模型依赖风险，同时根据任务复杂度、成本要求与质量需求自动调度最优模型，实现 "质量 - 成本 - 速度" 的最佳平衡。

技术选型依据与核心优势

- 采用通义千问（Qwen）与阶跃星辰（Minimax）双核心模型架构：Qwen 系列模型中文理解与结构化生成能力突出，推理成本低，适合基础备课任务；Minimax 多模态生成与长上下文能力更强，适合复杂教案编写与视频内容处理。

- 自研轻量级路由引擎，无需依赖复杂的模型网关，部署简单、维护成本低；支持动态配置模型权重与路由策略，可快速接入新模型。

- 内置模型降级与故障转移机制，当某一模型服务不可用时，自动切换至备用模型，保障核心业务连续性。

- 支持模型调用成本统计与用量管控，可按用户、项目、机构维度进行成本核算与限额管理。

项目具体应用

- 任务级路由：简单任务（如文本润色、标题生成）使用 Qwen-7B，常规备课任务使用 Qwen-14B，复杂任务（如跨学科教案、互动活动设计）使用 Minimax-abab6.5。

- 上下文路由：短上下文任务使用本地轻量级模型，长上下文任务（如整本教材解析）使用支持 128K 上下文的大模型。

- 成本路由：个人用户优先使用低成本模型，机构用户可根据需求选择高质量模型。

- 统一接口封装：所有模型调用通过统一接口暴露，上层业务无需关心底层模型差异，便于后续模型迭代与替换。

### 5.2.8 生成执行引擎（Diego Runtime）

自研Diego 生成执行引擎，专门针对教育场景的多步骤生成任务设计，解决传统单次大模型调用无法完成复杂备课流程的问题，实现生成过程的可编排、可追踪、可中断、可重试。

技术选型依据与核心优势

- 基于有向无环图（DAG）的流程编排能力，支持将复杂的备课任务拆解为多个原子步骤，灵活定义步骤之间的依赖关系与执行顺序。

- 内置状态管理与断点续跑机制，生成过程中系统崩溃或网络中断后，可从断点处继续执行，无需从头开始。

- 支持局部生成与增量更新，教师修改某一部分内容时，仅重新生成对应模块，避免整份课件重写。

- 提供生成过程的全链路可观测性，实时记录每个步骤的输入、输出与耗时，便于问题排查与生成质量优化。

项目具体应用

- 标准备课流程编排：将备课任务拆解为 "意图理解→大纲生成→大纲确认→内容分块生成→内容校验→格式美化→入库沉淀"7 个标准步骤，自动按顺序执行。

- 迭代修改流程编排：支持 "修改指令解析→定位目标模块→局部重生成→内容合并→版本更新" 的闭环流程。

- 批量生成任务编排：支持学期课件批量生成、多版本教材适配等批量任务的并行执行与进度监控。

- 自定义流程扩展：允许机构用户根据自身教学规范自定义生成流程，添加自定义校验步骤与内容模板。

### 5.2.9 预览渲染与导出链路（Pagevra + Artifact）

构建前端实时预览 + 后端标准化导出的完整内容呈现链路，解决传统生成工具预览与导出效果不一致、格式兼容性差的问题，确保生成的课件与教案可直接用于教学。

技术选型依据与核心优势

- Pagevra 前端渲染引擎：基于 Web 标准实现 PPT、教案的像素级实时预览，渲染效果与导出文件完全一致；支持局部内容的实时刷新，教师修改后可立即看到效果。

- Artifact 标准化导出引擎：完全基于开源工具链构建，无需依赖 Microsoft Office 或 WPS，避免商业软件授权问题与跨平台兼容性问题。

- 支持多格式统一导出：同一课程库内容可一键导出为 PPTX、DOCX、PDF、HTML5、思维导图等多种格式，所有格式保持内容与样式一致。

- 内置丰富的教学模板库，支持自定义品牌模板，机构可统一全校课件的版式与风格。

项目具体应用

- 实时预览：教师在生成过程中可随时预览课件与教案效果，支持翻页、缩放、全屏查看。

- 局部修改预览：教师提出修改意见后，仅重新渲染修改部分，预览响应时间小于 1 秒。

- 标准化导出：采用 Marp CLI 生成 PPTX 与 HTML5 课件，Pandoc 生成 DOCX 与 PDF 文档，导出文件可在所有主流办公软件中正常打开与编辑。

- 版本溯源：所有导出文件自动嵌入对应的课程库版本号与生成时间，便于追溯内容来源与更新记录。

### 5.2.10 项目空间语义服务（Ourograph）

自研Ourograph 项目空间语义引擎，作为课程资料库的核心语义层，负责管理课程知识的语义结构、关联关系与引用网络，实现从 "文件管理" 到 "知识管理" 的升级。

技术选型依据与核心优势

- 基于图数据库思想构建语义模型，将课程内容拆解为知识点、案例、习题等语义节点，建立节点之间的前置、关联、引用等语义关系。

- 支持跨库语义引用与知识复用，一个知识点可被多个课程库引用，一处更新所有引用处自动同步。

- 提供语义推理能力，可自动识别知识点之间的隐含关联，生成知识图谱与学习路径。

- 内置版本语义冲突检测机制，当上游库更新时，自动分析对下游派生库的影响，提示潜在的内容冲突。

项目具体应用

- 课程知识图谱构建：自动从课件、教案中提取知识点，生成课程知识图谱，可视化展示知识点之间的关系。

- 跨库引用管理：支持主基底库与辅助引用库的语义关联，自动同步上游库的知识点更新。

- 知识点关联推荐：在备课过程中，自动推荐相关的案例、习题与拓展资源。

- 知识溯源：支持每个知识点的来源追溯，显示其来自哪个文档、哪个版本，便于内容审核与质量管控。

### 5.2.11 API 契约治理（OpenAPI 模块化）

采用OpenAPI 模块化契约治理方案，实现前后端接口的统一管理与自动同步，同时为服务外包场景下的第三方系统集成提供标准化的接口规范。

技术选型依据与核心优势

- 基于 OpenAPI 3.0 标准定义 API 契约，采用模块化拆分方式，按业务域将接口划分为用户、课程、知识库、任务、导出等多个独立模块。

- 支持接口自动生成与类型同步：后端 FastAPI 自动生成 OpenAPI 文档，前端根据文档自动生成 TypeScript 类型定义与 API 调用代码，确保前后端接口一致性。

- 内置接口版本管理机制，支持多版本 API 并行运行，保证系统升级的向后兼容性。

- 提供完整的接口文档与示例代码，便于第三方系统集成与二次开发。

项目具体应用

- 模块化接口拆分：将系统 API 拆分为 12 个业务模块，每个模块独立定义、独立部署、独立升级。

- 前后端类型同步：使用 OpenAPI Generator 自动生成前端 API 客户端，避免手动编写接口代码导致的错误。

- 第三方集成接口：开放课程库管理、内容生成、资源导出等核心接口，支持与智慧校园、教务管理、学习平台等系统的无缝对接。

- 接口权限管控：每个 API 接口定义明确的权限要求，实现细粒度的接口访问控制。

### 5.2.12 部署与运行环境（Docker Compose + Nginx + SSL）

采用容器化一键部署方案，提供标准化的部署脚本与运行环境，支持公有云、私有云、本地服务器等多种部署方式，满足服务外包项目的快速交付与私有化部署需求。

技术选型依据与核心优势

- Docker Compose 容器编排：将所有服务组件打包为 Docker 镜像，通过一个 docker-compose.yml 文件定义整个系统的运行环境，实现 "一键部署、一键启动、一键升级"。

- Nginx 反向代理：负责请求分发、负载均衡、静态资源缓存与 SSL 终止，提升系统性能与安全性。

- 全链路 HTTPS 加密：所有外部访问均通过 HTTPS 协议传输，保障数据传输安全。

- 支持离线部署：可制作完整的离线安装包，在无互联网连接的教育专网环境中完成部署。

项目具体应用

- 单节点部署：提供标准化的单节点 Docker Compose 部署脚本，1-2 名技术人员可在 30 分钟内完成系统部署。

- 多节点扩展：当用户量增加时，可通过增加 Worker 节点的方式横向扩展系统性能，无需修改核心架构。

- Nginx 配置：配置静态资源缓存、请求限流、跨域处理与 SSL 证书，提升系统安全性与访问速度。

- 私有化部署支持：提供离线安装包与部署文档，支持学校本地服务器部署，所有数据完全存储在学校内部，满足数据安全与合规要求。
