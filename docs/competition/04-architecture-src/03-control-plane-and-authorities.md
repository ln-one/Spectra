# 03. 控制平面与六个正式能力层

## 本节要解释的系统问题

为什么 `Spectra Backend` 既不是空心网关，也不是大一统能力后端。

## 产品本体位置

- 主仓负责 workflow shell / orchestration kernel
- 六个正式能力层负责正式 authority
- clients / adapters 是反腐层，不是第二套领域实现

## 当前真实实现如何支撑

- 第 4 章和 canonical docs 已固定 six authorities 口径
- 第 5 章技术总图和第 8 章商业章都以六层能力支撑系统，而不是替代系统本体

## 可证明事实

- 运行现实中六个能力层已接入
- 第 2、4、5、8、9 章都已采用相同边界语义
