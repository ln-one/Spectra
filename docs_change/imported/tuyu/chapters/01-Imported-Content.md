# 2 项目综述
## 2.1 项目背景及意义
随着人工智能技术的迅速发展，自然语言处理（ NLP）领域取得了显著的进步，
通用型大型语言模型（ LLM）已经在许多任务上取得了令人瞩目的成果。一些开源的
大模型普遍是在百科、公共网络语料、开源代码库等数据上训练过的，知识分布虽然
很全面，但是在一个与公众网络差距较大的领域，这类语言模型并没有包含足够的领
域知识。在许多特定的垂直业务领域中，由于其与通用领域之间存在较大差异，直接
采用开源的通用型 LLM 经常无法满足该领域应用的要求。
旅游行业是一个涉及广泛、信息繁杂的领域，包括景点介绍、旅游规划、酒店预
订、餐饮推荐等多个方面，此领域类数据是一个相对容易获取且具备较高的数据价值。
本项目制作专用于旅游的垂直大模型，该模型能够针对旅游领域的特点进行深度优化，
通过大数据分析、自然语言处理等技术手段，实现对旅游资源的优化配置和旅游服务
的智能化升级， 围绕某一城市获取相关旅游数据并提供一个便捷的人机交互方式，提
升用户体验、促进多模态交互技术的发展，为旅游行业提供更加智能、高效的服务，
对于文旅产业的发展以及实现城市智能旅游助手等方面有着重要的意义。
## 2.2 项目目标
贯彻落实科技部等六部门联合印发的《新一代人工智能发展规划》 、 《关于加快场
景创新以人工智能高水平应用促进经济高质量发展的指导意见》等文件精神，创新建
设基于旅游垂直大模型的语料库问答系统，提升大模型文本生成、语义理解和用户交
互等方面的能力，为旅游行业提供更加智能、高效的服务。
通过网络、书籍等方法整理收集目标城市的文旅数据，包括但不限于城市的历史、
名人、景点、饮食特色、热门店铺等信息，构建较为全面完整的语料库；基于 Vue、N
uxt 框架进行用户交互界面设计，搭建可视化呈现的网站平台，实现具有多样化功能特
性的前端用户界面，同时基于 Python、Django 技术设计管理服务端，最终实现基于大
模型的检索数据、自然语言智能理解和自动生成的语料库问答系统，形成功能完备、
运行流畅的智能化人机交互平台。

务的质量和效率，垂直大模型项目有助于提升旅游行业的整体形象和竞争力，推动旅

### 5.2.1 Langchain
在这份技术文档中，我们会对 LangChain 这一框架进行详细的介绍。LangChain 是
一个基于语言模型（Language Model, 简称 LM）的开发框架，它通过串联不同的模型
和工具，使得开发者可以构建出能够理解和生成自然语言的应用程序。
 LangChain 的特点包括：
(1) 上下文感知：能够理解并处理输入数据的上下文信息，这包括但不限于用户的
提示指令、示例和背景资料等。这样可以让语言模型更加精准地捕捉到用户的需
求。
(2) 推理能力：LangChain 利用语言模型的推理能力来解释和响应用户的输入。这
不仅仅是简单地匹配模式或关键字，而是通过理解输入的内容，在给定的上下文
中做出恰当的回应或决策。
 LangChain 在本系统中的应用结合了三种类型的模型：
(1) 对话模型：处理和维护对话流，确保连贯性和上下文的一致性。
(2) 向量模型：用于理解和编码语义信息，支持模糊匹配和意图识别。
(3) 重排模型：对可能的回答或动作进行排序，选择最佳选项作为响应。

HNSW 的介绍:HNSW 是一种基于图的近似最近邻搜索方法，由 Yury Malkov 和 D
mitry Yashunin于 2016 年提出。它是一种高效的向量索引方法，旨在快速找到高维空间
中的最近邻元素。
HNSW 的工作机制基于“小世界”网络的概念，即大多数节点可以通过相对较少的
跳数被高效地达到。HNSW 构建了一个分层的图结构，每个层次都是一个小世界图。
最上层的图含有较少的节点，并覆盖了整个向量空间；而每向下一层，节点数量增多，
图的密度也随之增加。搜索时，算法先在上层的稀疏图中快速定位，然后逐层向下，
直到达到最底层的密集图，从而找到最近邻。

(2) ef_construction ：用于构建图形的动态候选列表的大小。在算法遍历图形的
过程中，这个列表会暂时保存迄今为止发现的最接近的候选点。一旦完成了对某

# 5 向量相似性检索技术
向量相似性检索是一种尖端技术，它在大规模的稠密向量数据集中寻找与指定查
询向量在语义上最为接近或相关的前 k 个结果。这种技术被广泛应用于多种数据类型，
如文本、图像、视频和音频等。为了执行相似性搜索，首先需要将这些非结构化数据
通过向量模型转换成高维数值向量，即我们所说的嵌入向量。
在生成嵌入向量之后，向量搜索引擎便开始工作，通过比较输入的查询向量与向
量存储库中的向量之间的空间距离，来寻找相似性。两个向量在空间中的接近程度越
高，它们的相似度就越大，从而表明这两个实体在语义上或特征上的接近程度。
向量相似性搜索的核心概念涉及到索引构建和距离度量两个方面。之前的段落已
经对索引做了阐述，这里我们聚焦于距离度量，它是评估向量之间相似度的关键。主
要存在三种距离度量方法：L2 距离（欧几里得距离） 、余弦相似度和内积。
在我们的系统中，我们选用 L2 距离来衡量两个向量之间的距离，这也是一种广泛
应用的方法。L2 距离，或称为欧几里得距离，通过计算两个点在各维度上差值的平方
和，然后取其平方根来确定。具体的数学表达式为：

# 7 获取图片的base64 编码
 base64 编码的获取
Base64 是网络上最常见的用于传输 8Bit 字节码的编码方式之一，Base64 就是一种
基于 64 个可打印字符来表示二进制数据的方法。可查看 RFC2045～RFC2049，上面有
MIME 的详细规范。Base64，就是包括小写字母 a-z、大写字母 A-Z、数字 0-9、符号"+
"、"/"一共 64 个字符的字符集， （任何符号都可以转换成这个字符集中的字符，这个转
换过程就叫做 base64 编码。Base64 编码是从二进制到字符的过程，可用于在 HTTP 环
境下传递较长的标识信息。采用 Base64 编码具有不可读性，需要解码后才能阅读。[4]
对于 CSS 的前端网页来说，嵌入 base64 编码后的图片的方式相对固定，即采用 backgro
und-image: url(data:image/png;base64,base64 编码);的格式，故而采用正则表达式可以迅
速获取 base64 的编码，正则表达式的字符匹配为：'background-image: url\("data:image/p
ng;base64,(.*)"\);'。
 base64 的解码
本项目采用 python 中的 base64 包的 b64decode 方法进行解码，然后保存图片于本
地。由于 b64decode 只能对长度为 4 的倍数的 base64 编码进行处理，故而在进行转换之
前需要进行预处理。首先，去除编码前后的 '换行符'、'缩进'以及'空格'。其次，如果编
码长度不为 4 的倍数，则使用'='进行填充，使其成为 4 的倍数。

# 11 测试和异常处理
 经过测试，存在一些异常问题：

## 6.3 模型的微调
通用 LLM 是在大型且多样化的数据集上进行培训的，而特定领域的 LLM 是专门
为特定领域设计的，并在满足组织特定要求的专有数据集上进行培训。通用 LLM 对许
多企业都有用，但特定领域的 LLM 更适合在特定领域内需要准确且适合上下文的输出
的情况。这些模型的性能通常优于作为 ChatGPT 基础的 GPT-3.5 等广泛模型，具有高
度针对性，并且可能更高效。
 但是在模型的微调过程中，有很多关键因素是需要我们考虑的，模型应该如何选
择?是选择基座模型还是 chat 对话模型?如何构建数据集？选择什么样的预训练方式？
选择什么样的微调方式？都是需要我们考虑的，所以我们下面将围绕这几个方面进行
讲解。
### 6.3.1 基座模型的选择
开源的 LLM 通常都会提供 base 和 chat 两种模型，首先明确 base 模型和 chat 模型
的区别：
 base 模型是在大规模语料上以 LM 这种形式的预训练任务（预测下一个 token）
训练得来的；
 chat 模型是在 base 模型上 sft+rlhf 上微调得来的。
但是选择 base 还是 chat 模型通常会影响到最后的模型微调后的效果，通过已有的
实验和数据表明，在数据量多的情况下，选择 base 模型可以更好的进行微调，并且模
型不会产生遗忘等效果。并且在 base 模型上进行微调，并不会破坏掉模型原本的通用
能力，从而实现完美的知识注入，我们选取的基座模型是 qwen-1.5-7B base 模型。
### 6.3.2 训练数据的构造
我们的数据要分别在预训练阶段和微调阶段分别对模型进行微调，但是这两个阶
段需要的数据格式不同。
# 1 预训练阶段：这一阶段是一个无监督学习过程，模型通过大规模数据自我学习，
而不是通过特定任务的标签引导。所以预训练阶段只需要 txt 格式的文本，且不需要特
定的 instruction，input，output 等特定标识.txt 格式的文本只需要把收集到的数据每块形
成 1 行即可。

### 6.3.4 LORA 微调
本项目是我们采取 LORA 微调的方式，首先来介绍一下 LORA 微调。
 微软和 CMU 的研究者指出，现有的一些 PEFT 的方法还存在这样一些问题：
(1) 由于增加了模型的深度从而额外增加了模型推理的延时，如 Adapter 方法。
(2) Prompt 较难训练，同时减少了模型的可用序列长度，如 Prompt Tuning 、Prefix
Tuning、P-Tuning 方法。
(3) 往往效率和质量不可兼得，效果差于 full-finetuning。
 有研究者对语言模型的参数进行研究发现：语言模型虽然参数众多，但是起到关
键作用的还是其中低秩的本质维度（low instrisic dimension） 。受到该观点的启发，提出
了 Low-Rank Adaption(LORA)，设计了如下所示的结构，在涉及到矩阵相乘的模块，引
入 A、B 这样两个低秩矩阵模块去模拟 Full-finetune 的过程，相当于只对语言模型中起
关键作用的低秩本质维度进行更新。

途语——文旅AI 对话系统  |  项目详细方案
# 6 untitled

图 6-27  LORA 微调
 LORA 的优点如下：
(1) 相比于原始的 Adapter 方法“额外”增加网络深度，必然会带来推理过程额外的
延迟，该方法可以在推理阶段直接用训练好的 A、B 矩阵参数与原预训练模型的参
数相加去替换原有预训练模型的参数，这样的话推理过程就相当于和 Full-finetune
一样，没有额外的计算量，从而不会带来性能的损失。
(2) 由于没有使用 Prompt 方式，自然不会存在 Prompt 方法带来的一系列问题。
(3) 该方法由于实际上相当于是用 LoRA 去模拟 Full-finetune 的过程，几乎不会带来
任何训练效果的损失。

## 9.1 基于PEST 的市场分析
### 9.1.1 政策环境（P）
文旅产业垂直应用的大语言模型，是文旅产业新兴的“智慧旅游”领域，在近年
来受到国家的高度重视。为推动 AI 技术的落地应用，促进行业的高新发展，政府颁布
了一系列针对性强、操作性高的政策，旨在推动文旅产业的数字化转型和创新发展。
国务院在 2021 年 12 月 22 日发布了《 “十四五”旅游业发展规划》 ，明确提出利用大数
