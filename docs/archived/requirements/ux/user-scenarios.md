# 用户场景（简版）

> 更新时间：2026-03-02
> 仅保留与当前 MVP 和近期迭代相关的核心场景。

## 场景 1：快速生成可讲授课件

- 用户：一线教师
- 目标：在短时间内完成可讲授的 PPT + 教案
- 主流程：
  1. 登录并创建项目
  2. 填写主题与教学目标
  3. 发起生成并下载结果
- 验收标准：
  - [ ] 可在 5 分钟内完成流程
  - [ ] 下载文件可打开并可继续编辑

## 场景 2：基于参考资料生成

- 用户：有现成教学材料的教师
- 目标：复用已有 PDF/Word/PPT 内容
- 主流程：
  1. 上传资料
  2. 系统解析并建立检索内容
  3. 生成课件并核对关键内容
- 验收标准：
  - [ ] 资料可成功解析
  - [ ] 生成内容能引用上传资料主题

## 场景 3：迭代修改与再生成

- 用户：需要多轮打磨课件的教师
- 目标：根据反馈快速调整课件内容
- 主流程：
  1. 查看预览
  2. 提交修改指令
  3. 获取新版本并对比
- 验收标准：
  - [ ] 修改指令可被系统执行
  - [ ] 新旧版本可追踪

## 非目标（当前阶段）

- 复杂协作编辑
- 大规模教学资源运营
- 全自动多模态深度理解（视频/语音全链路）
