# 前端导向设计：合理性评估与校正

## 1. 评估结论

整体方向合理：
- 交互目标清晰（配置 -> 共创 -> 预览）。
- 状态机明确，适合契约优先推进。
- 与“会话化任务系统”天然匹配。

需要校正的点：
- 原文偏视觉叙事，缺少失败与降级规则。
- “重写大纲”缺少可执行动作定义（已补 redraft 契约）。
- 动画描述较重，需明确性能降级与可中断约束。

## 2. 已完成的校正

- 契约新增：`/outline/redraft`，闭合状态回路。
- 文档新增：冲突处理、能力协商、事件游标消费规则。
- 结构拆分：把超大文档拆为索引+专题，便于维护与审阅。

## 3. 实施边界（防返工）

必须先落地：
- 会话状态机正确性。
- 事件流与恢复语义。
- 大纲编辑与重写闭环。

可后置增强：
- 复杂视觉动效。
- 高阶画布选择与 AI 工具栏增强。

非目标（当前阶段）：
- 一次性覆盖全部多模态能力（视频理解、语音全流程）。
- 追求复杂动画而牺牲稳定性。
