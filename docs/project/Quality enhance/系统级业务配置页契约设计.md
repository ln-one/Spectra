# 系统级业务配置页契约设计

> 更新时间：2026-03-22
> 状态：设计中

## 1. 目标

本设计用于解决当前开发调试阶段频繁手改 `.env` 的问题。

第一版只做**系统级业务配置**，不做项目级配置，不做会话级配置，也不做在线编辑部署环境变量。

## 2. 设计原则

1. 只管理业务运行配置，不管理敏感和部署级配置。
2. 复用现有后端主链路和运行时配置体系，不新建平行配置系统。
3. 第一版优先服务开发调试和前端联调。
4. 配置修改后全局生效，至少影响后续新请求。

## 3. 不在本页管理的内容

以下内容继续保留在 `.env` 或部署系统中：

- 数据库连接
- JWT 密钥
- 各类 API Key
- Redis / Stratumind / Qdrant / 存储路径
- 容器、端口、CORS 等部署配置

## 4. 第一版配置范围

### 4.1 模型相关

- `default_model`
- `large_model`
- `small_model`

页面直接展示真实模型名，不做抽象别名。

### 4.2 生成默认值

- `default_output_type`
- `default_page_count`
- `default_outline_style`

### 4.3 功能开关

- `enable_ai_generation`
- `enable_file_upload`
- `feature_flags`

### 4.4 体验与超时

- `chat_timeout_seconds`
- `ai_request_timeout_seconds`

## 5. 契约建议

### 5.1 读取

`GET /api/v1/system-settings`

返回结构建议：

```json
{
  "success": true,
  "data": {
    "models": {
      "default_model": "qwen3.5-flash",
      "large_model": "qwen3.5-plus",
      "small_model": "qwen3.5-flash"
    },
    "generation_defaults": {
      "default_output_type": "PPT",
      "default_page_count": 12,
      "default_outline_style": "structured"
    },
    "feature_flags": {
      "enable_ai_generation": true,
      "enable_file_upload": true
    },
    "experience": {
      "chat_timeout_seconds": 90,
      "ai_request_timeout_seconds": 60
    }
  },
  "message": "获取系统配置成功"
}
```

### 5.2 更新

`PATCH /api/v1/system-settings`

请求结构建议：

```json
{
  "models": {
    "default_model": "qwen3.5-flash",
    "large_model": "qwen3.5-plus",
    "small_model": "qwen3.5-flash"
  },
  "generation_defaults": {
    "default_output_type": "PPT",
    "default_page_count": 12,
    "default_outline_style": "structured"
  },
  "feature_flags": {
    "enable_ai_generation": true
  },
  "experience": {
    "chat_timeout_seconds": 90
  }
}
```

## 6. 后端职责

C 负责这一块的后端实现，包括：

- 配置模型
- 配置持久化
- 配置接口
- 运行时读取逻辑

## 7. 前端职责

B 负责配置页界面与交互，不负责定义配置语义。

## 8. 当前阶段取舍

第一版先不做：

- 权限细分
- 项目级覆盖
- 会话级覆盖
- 配置变更审计系统

先把“能改、能读、全局生效、前后端联调顺”做出来。
