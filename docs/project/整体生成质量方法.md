# 以 PPT 为重点、带动所有生成质量提升的整体质量方法

> 更新时间：2026-03-25
> 状态：当前生效
> 定位：成员 D 当前阶段的统一质量方法

## 1. 方法目标

当前阶段不再把“主链路是否存在”作为 D 的重点，而把重点放在：

1. 用 `PPT` 作为最直观、最容易暴露问题的主锚点
2. 把在 `PPT` 上验证过的方法迁移到 `大纲 / 卡片 / Word`
3. 让质量工作沉淀成可复现的 `样本 + baseline + 对照 + 结论`

因此，这套方法不是单独服务 PPT，而是以 PPT 为切入口，带动整个生成体系的质量收口。

## 2. 为什么先抓 PPT

优先以 PPT 为重点，原因是：

1. PPT 同时暴露结构、表达、信息密度、图文比例、素材命中等复合问题
2. PPT 最接近评委和用户的直接观感，最适合做前后对照
3. 一旦 PPT 质量方法成立，其中的大部分规则可以复用到大纲、卡片和文档生成

当前已具备的基础：

1. `backend/eval/ppt_quality_audit.py`
2. `backend/eval/ppt_quality_baseline.py`
3. `backend/eval/ppt_quality_comparison_audit.py`
4. `backend/eval/baselines/ppt-quality-baseline-v1.json`

当前基线事实表明，PPT 质量工作的核心不是“有没有能力”，而是“整体通过率仍然偏低，需要持续优化”：

1. `overall_pass_rate = 0.1667`
2. `information_density_pass_rate = 0.6667`
3. `visual_balance_pass_rate = 0.6667`
4. `image_match_pass_rate = 0.6667`

这说明 PPT 是最适合牵引质量方法的主锚点。

## 3. 统一质量闭环

整体质量工作默认按以下闭环推进：

1. 先收集真实问题样本
2. 再定义可重复运行的 baseline
3. 再补提示词与结构规则
4. 再做优化前后对照
5. 最后输出可稳定接入主线的建议与技术结论

不允许跳过样本和 baseline，直接凭主观感觉调 prompt。

## 4. 从 PPT 迁移到其他生成能力

### 4.1 迁移到大纲

PPT 上暴露出的“结构不清、重复、表达空泛”问题，会直接回流到大纲侧。

因此迁移顺序是：

1. 先在 PPT 中识别结构与表达问题
2. 回溯到大纲是否存在重复标题、重复知识点、页间同义反复
3. 用大纲专项 baseline 验证是否改善

当前已具备的大纲基础：

1. `backend/eval/outline_quality_audit.py`
2. `backend/eval/outline_quality_baseline.py`
3. `backend/eval/baselines/outline-quality-baseline-v1.json`
4. `docs/project/大纲去重与质量提升方案.md`

### 4.2 迁移到卡片

卡片质量不单独另起一套方法，而是复用 PPT 的主逻辑：

1. 看结果是否可演示
2. 看结构是否可用
3. 看资料与素材是否被真实利用
4. 看优化前后是否能形成对照

因此卡片优化默认优先处理更容易形成质量差异的卡片：

1. `interactive_quick_quiz`
2. `knowledge_mindmap`
3. `demonstration_animations`
4. `speaker_notes`
5. `classroom_qa_simulator`

对应方案见：

1. `docs/project/卡片生成质量优化方案.md`

### 4.3 迁移到 Word / 其他输出

Word 不单独作为第一优先级，但会复用以下质量方法：

1. 结构完整性
2. 信息密度控制
3. 教学表达清晰度
4. 来源与资料利用度

也就是说，PPT 是主锚点，但不是唯一受益对象。

## 5. 与检索、提示词、图片命中的关系

生成质量默认分成三层：

1. `检索 / 重排 / 图片命中`：决定有没有拿到对的资料和素材
2. `提示词 / 结构规则`：决定是否把资料组织成可用结果
3. `对照与 baseline`：决定能否证明优化确实有效

这三层必须同时存在，不能只做其中一层。

当前已具备的证据基础包括：

1. `backend/eval/baselines/rag-baseline-v1.json`
2. `docs/project/检索提示词图片命中评测口径.md`
3. `docs/project/PPT图片召回与素材命中优化方案.md`
4. `docs/project/PPT插图策略方案.md`

## 6. 当前阶段的默认执行顺序

成员 D 当前默认按以下顺序推进：

1. 先做 PPT 质量样本、baseline 和对照
2. 再把方法迁移到大纲重复质量
3. 再做卡片质量专项
4. 再整合检索、重排、提示词、图片命中证据
5. 最后给出可接入主线的建议和初审可用结论

## 7. 最终输出形态

如果这套整体质量方法生效，默认应至少产出：

1. 一组问题样本池
2. 一套 baseline
3. 一组优化前后对照
4. 一组可接入主线的规则建议
5. 一份可写入初审材料的技术结论

## 8. 约束

1. 不把 D 的质量方法写成对 C 主链路的替代实现
2. 不把“主观觉得更好”当作完成标准
3. 不把单次成功样例当作稳定质量结论
4. 所有结论尽量落在样本、baseline、对照和回归之上
