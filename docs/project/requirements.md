# 项目需求文档

> 本文档来自比赛原始需求，记录项目的核心目标和功能要求。

## 项目背景

当前AI辅助教学工具存在的问题：
- 功能单一、操作割裂，未形成以教师教学思路为核心的闭环工作流
- 教师需耗费大量时间在课件的内容组织、格式调整、资源搜寻等重复性劳动
- 工具只能执行单次简单指令，无法通过深度对话理解教师复杂的教学思路

## 项目目标

开发一个多模态AI互动式教学智能体，构建以教师教学思路为驱动、具备深度互动与多模态解析与生成能力的课件共创系统。

## 核心功能要求

### 1. 理解意图
- 通过自然语言（语音/文字）与教师进行多轮对话
- 主动询问、确认细节
- 清晰、完整地理解教师的教学目标、核心知识点、讲授逻辑、重点难点、互动设计思路等

### 2. 融合多模态参考
- 允许教师上传多种格式的参考资料（PDF教案、Word文档、参考视频片段、图片等）
- 从参考资料中提取关键信息（知识结构、案例、排版风格）
- 将提取的信息融入课件生成过程

### 3. 生成课件初稿
- 基于深度理解后的教学意图和参考资料
- 生成结构完整、内容丰富的多模态课件初稿
- 包括PPT演示文稿、Word教案文档
- 按教师要求生成知识点动画创意、互动小游戏等

### 4. 支持迭代优化
- 教师可基于预览提出修改意见
- 智能体理解修改要求并进行调整
- 形成"互动-生成-反馈-再生成"的闭环优化流程
- 支持下载后自行修改

## 技术实现要求

### 1. 本地知识库RAG
- 收集本专业知识库资料，作为大模型检索增强RAG的知识库输入
- 用大模型检索增强技术实现文本的处理、向量化、检索

### 2. 多模态交互式需求输入界面
- 提供语音输入和文字输入两种输入方式
- 实现智能对话能力：主动发起提问以澄清模糊需求（教学目标、知识要点、时长、产出风格等）
- 支持多轮对话，并能总结确认最终需求
- 提供参考资料上传功能（支持PDF、Word、PPT、图片、视频等）
- 参考资料与教师输入的意图需有对应关系

### 3. 教学意图理解与知识融合模块
- 利用大模型技术，理解教师输入的自然语言描述和对话历史
- 结构化提取教学要素（知识点清单、逻辑顺序、重点难点等）
- 对上传的参考资料进行内容解析（文本提取、视频关键帧分析或摘要生成）
- 将教师意图与参考资料信息、本地知识库信息进行有效融合
- 形成详细的课件生成指令集

### 4. 多模态课件生成引擎
- **PPT生成**：根据指令集，自动生成包含封面、目录、内容页、总结页的PPT课件，内容页应图文并茂，逻辑层次清晰
- **Word教案生成**：生成与PPT配套的详细教案，包括教学目标、教学过程、教学方法、课堂活动设计、课后作业等
- **内容生成多样性**：应能根据教师要求生成知识点相关的动画创意、互动小游戏等（至少支持一种）

### 5. 迭代优化与导出功能
- 提供课件预览界面
- 教师可提出修改意见（如"调整顺序"、"简化某页"、"增加一个案例"）
- 智能体需能理解修改意见，并对课件进行调整再生
- 支持将最终满意的课件以标准格式（.pptx、.docx）下载
- 动画或小游戏可以以HTML5网页、GIF、MP4等导出或集成到PPT中

## 技术要求

### 必须项
1. **本地知识库**：知识库资料格式不限，若包含多媒体资料，需说明处理方案
2. **大模型**：至少使用一个大语言模型作为核心技术组件，用于意图理解、对话交互和内容生成
3. **多模态处理**：需演示对至少两种不同格式的参考资料（如PDF和视频）的信息提取与利用能力
4. **实用性和创新性**：结合实际教学场景，过程易用、结果有效

### 技术选型
- 无限制，自行选择
- 可以基于开源的大模型和相关算法

## 项目价值

1. **减负增效**：将课件制作时间从数小时缩短至分钟级
2. **思路聚焦**：教师可将精力完全集中于教学设计和内容质量本身
3. **个性化满足**：智能体能够充分理解并实现其独特的教学风格和特定要求
4. **提升质量**：通过融合优质参考资料和AI的创造性，产出内容更精准、形式更生动的课件
5. **促进创新**：降低复杂互动、动画等形式的设计成本，鼓励教师尝试更多元化的教学方法
