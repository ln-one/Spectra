# PPT 插图可接入规则包

> 更新时间：2026-03-27
> 状态：当前生效
> 定位：成员 D 提供给成员 C 的 PPT 插图主链路接入输入

## 1. 目的

本规则包不是由 D 直接接入主链路，而是提供给 C 的稳定输入。

它回答 4 个必须明确的问题：

1. 哪些页面值得插图
2. 图片放在页面什么位置更稳
3. 一页应放几张图
4. 哪些页面有较高版式风险，应默认不插图

当前规则包严格服务于现阶段的 `PPT` 生成，不扩展到独立卡片或其他导出链路。

## 2. 设计依据

本规则包的组织方式参考以下资料中的共同原则：

1. [提示词工程参考资料清单.md](提示词工程参考资料清单.md)
2. OpenAI Prompt Engineering Guide
3. Anthropic Prompt Templates and Variables
4. Paper2Poster
5. TeachMaster / Generative Teaching via Code

当前采用的原则是：

1. 先把“任务语义”和“页面约束”写清，再让模型决定是否插图。
2. 先给出页级决策规则，再给图位和数量规则。
3. 让 prompt 输出遵守稳定结构，而不是用模糊自然语言自由发挥。
4. 如果素材命中不足或版式风险过高，默认不插图。

## 3. 选图页规则

### 3.1 优先插图页

以下页面默认优先考虑插图：

1. 过程讲解页
2. 结构讲解页
3. 实验现象页
4. 并列对比页
5. 明显依赖观察、空间关系或动态变化才能降低理解门槛的页面

### 3.2 默认不优先插图页

以下页面默认不优先插图：

1. 纯概念定义页
2. 结论总结页
3. 已经文字较满的页面
4. 没有高相关素材的页面
5. 图片无法直接支撑当前页面核心结论的页面

### 3.3 选图页评分口径

单页是否值得插图，默认按以下四项判断：

1. `page_semantic_fit`
   - 当前页是否属于过程、结构、实验、对比等图示受益明显的类型
2. `keyword_coverage`
   - 检索到的素材是否覆盖本页标题或核心结论关键词
3. `teaching_dependency`
   - 教师讲解是否需要借助视觉对象指向“图上看什么”
4. `layout_capacity`
   - 页面是否有足够空间容纳图片而不破坏主要信息结构

只有以上四项整体满足时，才应进入插图候选。

## 4. 图位规则

### 4.1 优先图位

1. 右侧并列
   - 适合 2-3 组要点 + 1 张结构图 / 流程图
2. 左侧并列
   - 适合需要先看图、后看解释的页面
3. 下方承接
   - 适合标题 + 少量要点 + 1 张横向示意图

### 4.2 谨慎图位

1. 页尾追加
   - 只有在正文已足够短且不会造成分裂阅读时才允许
2. 顶部大图
   - 只有在图片承担主要观察任务、文字能压缩到 2-3 个要点时允许

### 4.3 默认避免

1. 文本已经较密时仍强行并列图片
2. 图片占位后导致正文必须缩成碎片化短句
3. 同页出现两个视觉焦点，教师讲解无法明确主次

## 5. 数量规则

默认数量策略如下：

1. `0` 图
   - 概念定义页
   - 结论总结页
   - 无高相关素材页
   - 版式容量不足页
2. `1` 图
   - 当前默认优先策略
   - 适用于绝大多数过程、结构、实验说明页
3. `2` 图
   - 仅用于显式前后对比、双对象并列对照
   - 且要求文字能进一步压缩、页面主次仍清楚

当前阶段默认不建议超过 `2` 图。

## 6. 版式风险规则

以下情况应判为高风险，默认不插图或降回 `0` 图：

1. 图片会挤压掉本页必要文本空间
2. 插图后页面信息重心失衡
3. 图片与文字分别承担不同结论，造成图文不一致
4. 图片存在，但教师无法在讲解中明确指出“图上看什么”
5. 插图让页面从“可讲”退化成“只能看”

## 7. 接入要求

若 C 后续将本规则包接入主链路，建议最少保留以下决策元数据：

1. `retrieval_mode`
   - `default_library` 或 `strict_sources`
2. `page_semantic_type`
3. `image_insertion_decision`
   - `insert` / `skip`
4. `image_count`
5. `image_slot`
   - 如 `left_split` / `right_split` / `bottom_panel`
6. `layout_risk_level`
   - `low` / `medium` / `high`
7. `image_match_reason`
8. `skip_reason`

## 8. 当前边界

1. 本规则包由 D 输出，不代表 D 负责主链路接入。
2. 本规则包用于收口当前图片插入语义，不替代 C 的运行时实现。
3. 后续实验和 baseline 结果必须同时标注检索语义：
   - 不传 `rag_source_ids`：默认库检索
   - 传 `rag_source_ids`：严格限源检索
