# 04. 三条核心主链

## 本节要解释的系统问题

为什么系统不是“很多流程并列”，而是三条会互相咬合的主链。

## 产品本体位置

- 资料进入与证据组织链
- Session 生成与交付链
- 知识回流与演化链

## 当前真实实现如何支撑

- 当前主链现实已固定为 `Session -> Diego -> Pagevra -> Ourograph`
- 第 5 章和第 6 章已经分别承担“机制说明”和“证据验证”
- 第 4 章负责把它们先收成系统设计判断

## 可证明事实

- 上传/检索/生成/预览/导出/绑定已经在主稿其他章节中形成一致叙事
- 第 4 章需要先把这三条主链讲清楚，后续章节才不至于散
