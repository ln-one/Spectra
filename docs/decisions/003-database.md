# ADR-003: 数据库选择 - SQLite (初期)

**状态**: ✅ 已接受  
**日期**: 2026-02-15

## 背景

需要选择数据库来存储用户数据、对话历史、项目信息等。

## 考虑的方案

1. SQLite
2. PostgreSQL
3. MySQL
4. MongoDB

## 决策

初期使用 **SQLite**，后期可迁移到 PostgreSQL

## 理由

- 零配置，开发简单
- 足够满足初期需求
- Prisma ORM 支持无缝迁移
- 简化部署流程

## 权衡

- 并发写入性能有限
- 不适合大规模生产环境
- 但初期完全够用

## 影响

- 开发阶段使用 SQLite
- 需要规划后期迁移方案
- 数据库设计要考虑可移植性
