## 项目目标

### 要解决的核心问题

`Spectra` 当前阶段主要解决以下四个问题：

1. 让教师在一个工作台中完成资料整理、会话推进和结果生成，减少工具来回切换；
2. 让上传资料进入解析、检索增强和证据组织流程，避免停留在附件层；
3. 让生成过程可确认、可修改、可继续推进，避免一次性黑箱输出；
4. 让生成结果可以被保存、对比、引用和继续复用，避免导出后即与系统失联。

### 目标与当前作品对应关系

| 目标 | 当前作品落点 | 当前证据 |
| --- | --- | --- |
| 在同一工作台中完成资料整理、会话推进和结果生成 | Studio 工作台、资料区、会话区、结果区、预览与下载页 | 第 5 章前端工作台截图、第 7 章界面展示、preview/download 接口 |
| 让上传资料进入后续解析、检索和来源查看 | 文件上传、远端解析、切块索引、检索增强、来源查看 | `POST /api/v1/files`、`POST /api/v1/rag/search`、文件上传索引测试、检索评估表 |
| 让生成过程支持确认、正式生成和继续修改 | `Session -> outline -> confirm -> generate -> refine` | 会话创建接口、Studio card 执行接口、preview contract 测试 |
| 让生成结果可以保存、查看历史、导出并继续复用 | `Artifact`、`Version`、`Reference`、history、preview/export/download | 结果绑定测试、项目语义测试、preview/export 接口 |

这些目标围绕同一套工作过程展开。教师先在工作台中组织任务和资料，上传内容进入解析与检索流程，再进入会话推进和正式生成，最终结果通过预览、导出和历史查看进入后续使用。每一项目标都能在当前页面、接口和测试中找到对应落点。

第一层目标对应统一工作台。教师不需要在多个独立工具之间来回切换，而是可以在同一界面内完成资料进入、会话推进、结果查看和结果导出。第二层目标对应资料处理与检索增强。上传资料不会停留在附件层，而会继续进入解析、切块、索引和来源查看。第三层目标对应生成过程控制。系统先形成 `outline`，再进入确认、正式生成和后续修改。第四层目标对应结果管理。结果不会在导出之后脱离系统，而会继续保留为可查看、可比对、可继续使用的正式对象。

从当前作品完成情况看，项目目标已经与系统结构形成直接关系。前端工作台解决入口分散的问题，知识库处理与检索增强解决资料无法进入后续流程的问题，会话推进解决生成过程不可控的问题，结果对象与版本关系解决导出后难以继续修改的问题。后续章节中的模块设计、核心技术和测试结果，都是对这四项目标的展开说明。
