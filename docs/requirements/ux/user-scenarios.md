## 多模态交互细节（Multimodal Nuances）

> 本节专门描述多模态内容的智能处理和交互设计

### 1. 视频处理交互

#### 1.1 视频上传与解析
**自动处理流程**：
```
上传视频 → 关键帧提取 → 场景识别 → 时间轴标注 → 用户选择
```

**关键界面要点**：
- 支持视频预览与时间轴分段标注（引入/演示/互动/总结）
- 自动生成关键帧列表并标注推荐帧
- 支持对关键帧进行勾选、预览、用途说明

**用户操作**：
- **选择时间段**：拖动时间轴选择"01:15-03:30"
- **选择截图**：勾选图3和图5
- **指定用途**：
  - "作为第3页背景"
  - "生成分步演示（4页）"
  - "提取为独立图片素材"

#### 1.2 视频截图智能处理
**自动去背景**：
- 识别主体内容（如实验器材、演示物体）
- 自动去除杂乱背景
- 保留关键元素

**自动排版**：
- 根据PPT主题色调整图片色调
- 智能裁剪和缩放
- 添加阴影和边框效果

---

### 2. PDF处理交互

#### 2.1 PDF结构化解析
**智能识别**：
- 标题层级（一级标题、二级标题）
- 正文段落
- 公式和符号
- 表格和图表
- 练习题和答案

#### 2.2 复杂公式保持格式
**LaTeX识别与转换**：
```
PDF中的公式：F = ma
         ↓ OCR识别
LaTeX格式：F = ma
         ↓ 渲染
PPT中的公式：F = ma (矢量格式，可缩放)
```

**交互选项**：
- **保持原样**：直接截图插入
- **重新渲染**：转为LaTeX后重新排版
- **简化显示**：转为纯文本

---

### 3. 语音输入的教学术语纠错

#### 3.1 学科专业词库
**预置词库**：
- 数学：分数、方程、几何、函数...
- 物理：牛顿、力学、电磁、光学...
- 化学：元素、反应、分子、原子...
- 英语：语法、时态、词汇、句型...

**智能纠错示例**：
```
用户语音："我要讲牛顿第二定理"
         ↓ 识别
初步结果："我要讲牛顿第二定理"
         ↓ 术语纠错
最终结果："我要讲牛顿第二定律"
         ↓ 提示
系统："已自动纠正为'牛顿第二定律'（物理学定律）"
```

#### 3.2 上下文智能判断
**场景识别**：
```
项目信息：高二物理
用户语音："F等于M乘A"
         ↓ 上下文分析
识别为：F = ma（牛顿第二定律公式）
         ↓ 自动转换
显示为：F = ma（带公式格式）
```

---

### 4. 本地知识库RAG

#### 4.1 用户价值
- 教师可以沉淀校本资料、历年教案、学科题库，减少重复备课
- 生成内容可追溯到本地知识来源，降低“凭空生成”风险
- 在不同教师和课程之间复用高质量教学资产，提升一致性

#### 4.2 用户视角流程
```
导入本地知识资料（PDF/Word/PPT/题库） → 清洗切片 → 向量化入库
→ 生成时检索召回（Top-K） → 与教师意图融合生成 → 来源标注与可追溯
```

#### 4.3 基础交互要求
- 支持知识库资料上传、更新、删除、版本标记
- 支持按学段/学科/章节筛选检索范围
- 在生成结果中显示“命中来源”（文件名、章节、片段）
- 支持教师对检索片段进行“采纳/忽略/置顶”反馈

## 核心使用场景

### 场景1：从零开始创建新课件

**背景**：李老师需要准备下周的"分数加减法"课程

**使用流程**：

1. **启动对话**（5分钟）
   - 李老师打开系统，创建新项目"五年级数学-分数加减法"
   - 系统问候："您好！我是您的课件助手，请告诉我这节课的教学目标"
   - 李老师输入（或语音）："这节课要教五年级学生分数加减法，重点是同分母分数的加减"

2. **深度对话**（10分钟）
   - 系统主动提问：
     - "这节课计划多长时间？"（李老师：40分钟）
     - "学生之前学过分数的基本概念吗？"（李老师：学过）
     - "您希望用什么方式讲解？比如生活案例、图形演示等"（李老师：用切蛋糕的例子）
     - "需要设计课堂互动吗？"（李老师：需要，最好有练习题）
   - 李老师补充："我希望课件风格活泼一些，多用颜色和图片"

3. **上传参考资料**（5分钟）- 强化多模态交互
   
   **上传视频**：
   - 李老师上传了一个教学视频（其他老师的优秀案例，5分钟）
   - 系统开始智能解析（显示进度）：
     ```
     正在分析视频内容...
     ✓ 已识别3个教学片段
     ✓ 已提取8张关键截图
     ```
   - **侧边栏自动展示视频时间轴**：
     ```
     📹 教学视频.mp4 (5:23)
     ├─ 00:00-01:15 [引入环节] 生活情境导入
     ├─ 01:15-03:30 [核心演示] 切蛋糕动画演示 ⭐
     ├─ 03:30-04:45 [练习环节] 学生互动
     └─ 04:45-05:23 [总结环节] 知识点回顾
     
     [关键截图预览]
     [图1] [图2] [图3] [图4] [图5] [图6] [图7] [图8]
     ```
   - 系统询问："我识别到视频中有切蛋糕的动画演示（01:15-03:30），您想如何使用？"
   - **李老师可以直接操作**：
     - 勾选时间段："01:15-03:30"
     - 选择截图3和截图5
     - 指令："把这两张截图作为第3-4页的主图，保留动画演示的分步效果"
   - 系统确认："好的，我会将这段视频的关键帧转化为分步演示页面"
   
   **上传PDF**：
   - 李老师上传了一份PDF教案（10页）
   - 系统智能解析：
     ```
     正在分析PDF内容...
     ✓ 已识别文档结构
     ✓ 已提取5道练习题
     ✓ 已识别3个知识点
     ```
   - 系统展示结构化内容：
     ```
     📄 分数加减法教案.pdf
     ├─ 第1页：教学目标
     ├─ 第2页：练习题设计 ⭐ (5道题)
     ├─ 第3-5页：教学过程
     └─ 第6-10页：拓展内容
     ```
   - 系统询问："我在第2页发现了5道练习题，您想全部使用还是选择部分？"
   - 李老师点击预览："显示第2页"
   - 系统展示PDF第2页的预览
   - 李老师标注："使用前3道题，难度适合五年级"

   **上传PPT/Word参考**：
   - 李老师上传PPT模板（用于版式和配色参考）
   - 李老师上传Word讲义（用于教学目标与课堂活动表述参考）
   - 系统询问："这两份资料您希望优先参考结构、内容还是风格？"
   - 李老师选择："PPT参考风格，Word参考教学活动描述"

   - 系统自动检索校本知识库（五年级数学/分数章节）
   - 显示召回结果："命中《校本练习库》2条、《往年教案》1条"
   - 李老师选择："保留校本练习库第1条，忽略其余"

4. **确认需求**（2分钟）
   - 系统总结：
     ```

     - 主题：分数加减法（同分母）
     - 时长：40分钟
     - 年级：五年级

     - 用切蛋糕案例引入
     - 图形演示分数概念

     - 风格：活泼、多彩、图文并茂

     - 视频：切蛋糕动画演示
     - PDF：练习题设计

     ```
   - 李老师确认："正确，开始生成吧"

5. **生成课件**（2-3分钟）- 流式反馈
   - 系统显示详细的流式进度（不只是进度条）：
     ```
     ⏳ 正在分析参考资料...
     ✓ 已提取视频关键帧 (3张)
     ✓ 已解析PDF练习题 (3道)
     ✓ 已召回本地知识库片段 (3条)
     
     ⏳ 正在生成课件结构...
     ✓ 已生成15页大纲
     
     ⏳ 正在生成PPT内容...
     ✓ 第1-3页：引入环节（使用视频截图）
     ✓ 第4-8页：核心知识点（图形演示）
     ⏳ 第9-12页：练习环节（转化PDF习题为互动题）
     ⏸ 第13-15页：总结与拓展

     ```
   - 生成完成，显示预览

6. **预览和修改**（10分钟）- 所见即所得 + 溯源
   - 李老师浏览PPT预览
   - **直接操纵交互**：
     - 李老师点击第3页的图片
     - 左侧对话框自动输入："我要修改第3页的图片..."
     - 李老师补充："太小了，放大一些"
     - 系统立即调整并更新预览
   
   - **内容溯源功能**：
     - 李老师点击第9页的练习题
     - 系统高亮显示："这道题来自您上传的PDF第2页第1题"
     - 李老师可以点击查看原始PDF内容
   
   - **智能建议**：
     - 李老师说："第5页的例题太难"
     - 系统分析："这道题涉及分数乘法，超出了五年级同分母加减法的范围"
     - 系统建议："我为您生成了3道符合五年级水平的例题，请选择：
       - 例题A：1/4 + 2/4 = ?（基础）
       - 例题B：3/5 + 1/5 = ?（标准）
       - 例题C：5/8 - 3/8 = ?（进阶）"
     - 李老师选择："用例题B"
   
   - **批量修改**：
     - 李老师："把所有页面的字体改大一点"
     - 系统："好的，已将正文字号从18pt调整为20pt"
     - 实时预览更新

7. **下载使用**（1分钟）
   - 李老师满意后，下载.pptx和.docx文件
   - 在本地微调后用于教学

**总耗时**：约30-35分钟（相比传统方式的3-4小时）

## 关键决策点

### 决策点1：输入方式选择
- **场景**：用户开始描述教学思路时
- **选项**：文字输入 vs 语音输入
- **影响因素**：
  - 用户打字速度
  - 环境是否安静
  - 思路是否清晰（语音更适合发散思维）
- **设计**：默认显示文字输入框，同时提供明显的语音按钮

### 决策点2：参考资料使用方式
- **场景**：用户上传文件后
- **选项**：
  - 整体参考（系统自动提取关键信息）
  - 指定参考（用户标注具体部分）
- **影响因素**：
  - 文件内容的相关性
  - 用户的明确程度
- **设计**：上传后主动询问，提供快速标注选项

### 决策点3：生成详细程度
- **场景**：确认需求后，准备生成
- **选项**：
  - 快速生成（基础版，5-10页）
  - 详细生成（完整版，15-25页）
- **影响因素**：
  - 课程时长
  - 用户需求的详细程度
- **设计**：根据对话内容智能推荐，用户可调整

### 决策点4：修改方式
- **场景**：用户对生成结果不满意
- **选项**：
  - 对话式修改（"把第3页改成..."）
  - 直接编辑（在预览界面点击修改）
- **影响因素**：
  - 修改的复杂度
  - 用户的技术水平
- **设计**：两种方式都支持，对话式为主

### 决策点5：本地知识库RAG使用策略
- **场景**：用户已上传参考资料，进入生成前
- **选项**：
  - 启用本地知识库增强（推荐）
  - 仅使用本次上传资料
- **影响因素**：
  - 是否有可复用的校本资料与历史教案
  - 教师对内容可控性和可追溯性的要求
- **设计**：默认启用RAG并展示检索来源，用户可按学段/学科/章节限制检索范围，支持逐条采纳或忽略

### 教学适配度目标（新增）

#### 1. 学段适配性
**目标**：AI生成的内容必须符合目标学段学生的认知水平

**评估标准**：
- 小学（1-6年级）：
  - 语言简单易懂，避免抽象概念
  - 多用图片、动画、游戏
  - 字号大（≥20pt），颜色鲜艳
  - 避免复杂公式和专业术语
  
- 初中（7-9年级）：
  - 语言规范，适度引入概念
  - 图文并茂，增加逻辑推理
  - 字号适中（18-20pt）
  - 可使用基础公式和术语
  
- 高中（10-12年级）：
  - 语言专业，深入分析
  - 重视逻辑和推导过程
  - 字号标准（16-18pt）
  - 使用完整的学科术语和公式

**智能检测示例**：
- 当检测到超学段概念（如五年级内容中出现“微积分”）时，系统给出预警并建议替代表述
- 教师可选择自动修正、保留原内容或手动修改

#### 2. 学情动态调整
**功能**：允许教师随时切换学生基础水平

**交互设计与效果**：
- 支持在“零基础/有基础/拔高”三种学情模式间切换
- 支持“应用到全部页面”或“仅当前页”
- 零基础模式偏重概念解释与生活案例；拔高模式偏重拓展与思考题

#### 3. 学科规范性
**目标**：确保内容符合学科教学规范和课程标准

**检查项**：
- 概念定义准确
- 公式表达规范
- 术语使用正确
- 逻辑推导严密
- 例题难度合理

**智能校验示例**：
- 自动检查知识点顺序、公式推导、例题梯度
- 发现难度跳跃时给出补充过渡例题建议

#### 4. 教学目标达成度
**评估维度**：
- 知识点覆盖完整性
- 重难点突出程度
- 互动环节设计合理性
- 时间分配科学性

**智能分析输出**：
- 按“知识点覆盖、重点突出、难点讲解、互动设计、时间分配”给出分项评分
- 提供针对性的改进建议（如补充辅助案例、增加总结回顾页）

#### 5. 个性化教学风格
**支持教师个性化需求**：
- 讲解风格可选：严谨型、生动型、互动型
- 课件风格可选：简约专业、活泼多彩、创意个性
- 支持保存为默认风格

**风格影响**：
- 生动型：多用比喻、故事、生活案例
- 严谨型：重视定义、推导、逻辑链条
- 互动型：多设计提问、讨论、小组活动
