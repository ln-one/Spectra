# 用户体验分析

> 负责人：成员B

## 目标

从用户（教师）的角度，分析他们如何使用系统，设计流畅的交互体验。

## 输出清单

- [ ] [用户场景分析](./user-scenarios.md)
- [ ] [交互设计](./interaction-design.md)
- [ ] [界面原型](./wireframes/)

## 关注点

1. **用户是谁**
   - 教师的背景、需求、痛点
   - 技术水平、使用习惯
   - 期望和目标

2. **使用场景**
   - 在什么情况下使用系统
   - 完整的使用流程
   - 关键决策点

3. **交互设计**
   - 核心交互流程（对话、上传、预览、修改）
   - 界面布局和导航
   - 反馈和提示

4. **界面原型**
   - 主要页面的原型
   - 可以是低保真线框图
   - 标注关键交互逻辑

## 建议

- 可以参考现有的教学工具
- 思考教师的真实工作流程
- 关注易用性和效率
- 考虑不同技术水平的用户

## 工具推荐

- Figma / Sketch（高保真原型）
- draw.io / Excalidraw（流程图）
- 纸笔（快速草图）
- 截图 + 标注

## 开始

创建以下文件并开始分析：
- `user-scenarios.md` - 用户场景分析
- `interaction-design.md` - 交互设计
- `wireframes/` - 界面原型（图片或链接）
