# 第 8 章源稿目录

> Status: `current`
> Role: internal source pack for `08-business-plan.md`.

本目录不是对外主稿，而是第 8 章“商业企划”的内部商业源稿。对外阅读仍以 `08-business-plan.md` 为准，本目录负责深写商业逻辑、平台生态、商业边界、未来演进和风险控制，避免第 8 章在一个文件里越写越散。

## 使用规则

- 对外主章必须保持商业方案书口气，不能像内部备忘录。
- 本目录可以记录更细的商业推演、阶段边界、写作禁区和后续扩写素材。
- 当前已实现能力与未来商业演进必须分开写，不能把平台长期目标伪装成已经全部上线。
- 任何商业飞轮、收入层级、资产交易、学习者增值和机构私有化表述，都必须回到 `Project / Reference / Version / Artifact / CandidateChange / Member` 这套对象语言。
- 对外避免攻击性替代表述，统一使用“赋能创作者、放大课程资产、提升个性化学习、帮助机构治理知识资产”的表达。
- Mermaid 图应由源稿先定义，再汇总进主章；不要直接在主章里临时脑补商业结构图。

## 第 8 章主线

第 8 章不是“怎么收费”的小节，而是解释 `Spectra` 为什么具备平台型商业潜力：

- 课程库是资产节点；
- 权限组合和关系网络才是角色本体；
- 使用、引用、外化和演化会形成商业飞轮；
- 收入结构从工具订阅扩展到课程资产、学习者增值、组织平台和长期智能网络；
- 商业野心必须建立在当前产品面和架构现实之上，不能写成空泛愿景。
