# ADR-012 MVP 架构改进路线（简版）

- 状态：Accepted
- 更新时间：2026-03-02

## 目标

在不显著增加复杂度的前提下，提升 MVP 的安全性、一致性和可维护性。

## 已落地

- [x] 认证主链路
- [x] 项目/文件/生成任务核心接口
- [x] 统一错误响应与基础日志
- [x] 文档系统重构（MVP 对齐）

## 待补齐

- [ ] 非核心接口的权限一致性
- [ ] 幂等与限流覆盖面
- [ ] 文档解析可插拔（MinerU/LlamaParse）

## 交付顺序

1. 安全一致性（鉴权/越权/幂等）
2. 稳定性（错误处理/日志/回归测试）
3. 能力扩展（解析器和多模态）

## 非目标

- 不在 MVP 阶段引入重型分布式架构
- 不一次性切换全部基础设施
