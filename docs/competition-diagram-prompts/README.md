# Competition 图稿包

这是一套独立的 sidecar 图稿包，用来给外部图像生成系统提供结构骨架、中文主 Prompt 和统一风格约束。

## 使用说明

- `Mermaid` 不是成品图，只是结构说明书。
- 主受众是中国评委、中国老师、中文答辩与中文正文场景。
- 每个文件都可以被单独复制给外部系统直接使用。
- 主 Prompt 默认使用中文，英文只作为补充关键词。

## 版本定义

- `答辩版`
  - 用于 `PPT 横向`
  - 默认按 `16:9` 宽屏横向设计
  - 信息密度适当，不是极简空壳
  - 适合边讲边看，半页或整页横向插图
- `文档版本`
  - 用于 `Word 纵向`
  - 默认按 `A4 纵向` 设计
  - 信息密度更高，适合正文阅读和展示结构压强
  - 允许多一层内部逻辑，但不允许把字号压小
- `单版本`
  - 适合单一主场景，文件内会写明默认适配场景

## 核心原则

- 不是所有图都该稀，也不是所有图都该密。
- `答辩版` 不是“少信息版”，而是“横向讲解版”。
- `文档版本` 不是“小字堆砌版”，而是“纵向展开版”。
- 所有图都必须讲清楚：
  - 这张图只回答什么
  - 信息分几层
  - 外层分几组
  - 组内逻辑如何展开
- 所有图都必须维持：
  - 清晰 hierarchy
  - 明确 grouping
  - 中文可读
  - 统一低饱和专业风格

## 横版与竖版布局建议

- `分层架构图`
  - 答辩版：横向三段或横向三层
  - 文档版本：纵向三层或四层堆栈
- `闭环 / 主链图`
  - 答辩版：横向主链加关键回流
  - 文档版本：纵向主链或中心核加上下回流
- `产品能力簇图`
  - 答辩版：中心核加横向三簇
  - 文档版本：中心核加纵向三簇展开
- `商业 / 生态图`
  - 答辩版：更强中心、更快读
  - 文档版本：纵向分区和关系层展开

## 统一设计系统

- 气质：专业、高级、克制、低饱和、简约多彩
- 结构：卡片分组、细线连接、轻阴影、大留白
- 文字：中文短词、大标题、节点尽量单行
- 色板：
  - 控制层：灰蓝 / 雾青
  - 正式状态：灰绿 / 石墨蓝
  - 检索证据：浅青 / 湖蓝
  - 生成：低饱和暖橙点缀
  - 身份治理：冷灰 / 橄榄灰
  - 商业图：同一低饱和彩色体系
