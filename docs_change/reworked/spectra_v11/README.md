# Spectra 11 章重组稿

本目录用于承接 `Spectra.docx` 的第一轮重组改写。

当前目标不是直接产出终稿，而是先形成一版结构稳定、图文不断链、语气更稳的工作稿，方便后续继续按老师意见逐章精修。

## 当前产物

- `source/`：按新 11 章目录组织的工作稿 Markdown
  其中短章可继续使用单文件，长章已改为“章目录 + 子块文件”
- `anchors/`：按块组织的内部证据锚点，用于 AI 改写和人工核对，不进入正式导出稿
- `tone-balance-guide.md`：内部语气控制文档，用于平衡“作品证据”和“系统辨识度”，不进入正式导出稿
- `assets/media/`：从原始 `Spectra.docx` 自动继承的图片资源
- `figures.yaml`：图片与原章节、图题、相邻标题的映射信息
- `meta.yaml`：当前构建元信息
- `HANDOFF.md`：下一位接手者的最小工作说明
- `build/spectra-v11-draft.docx`：当前阶段导出的 Word 稿
- `build/spectra-v11-draft.merged.md`：构建前合并后的总 Markdown

## 当前写作原则

- 先按新目录重组，不急着逐句抠措辞
- 大章继续拆块，避免整章过大导致改写时上下文丢失
- 块级改写优先看对应锚点，按代码、接口、页面、测试和图片做事实核对
- 高层表达默认参考 `docs/project/` 中的哲学与需求收口文档，但只做点缀提炼，不直接搬运原文
- 少讲判断，多讲功能、页面、流程、模块和结果
- 需求、功能、实现、技术、效果尽量形成对应关系
- 图片优先跟随系统设计、核心技术、测试展示等关键章节迁移
- 参考《天眼》《途语》的章法，但不混入它们的产品事实
