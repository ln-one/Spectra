# Spectra 术语表

本文件用于统一服务外包大赛提交材料中的关键术语，避免前后章节表述不一致。

## 核心术语

### `LLM`（Large Language Model，大语言模型）

指具备自然语言理解、生成、总结和推理能力的大规模预训练模型。在 `Spectra` 中，LLM 主要用于意图理解、多轮对话、教学内容规划和课件文本生成。

### `RAG`（Retrieval-Augmented Generation，检索增强生成）

指在生成前先从知识库中检索相关内容，再将检索结果与用户问题一并提供给大模型，从而提高生成内容的准确性和可追溯性。

### `Multimodal Agent`（多模态智能体）

指能够同时处理文本、语音、图片、PDF、视频等多种输入形式，并可基于上下文主动做出理解、追问、生成和优化决策的智能系统。

### `Library`（库）

指系统对外的核心业务对象，本质上是一个课程数据库节点。它不是单个 `PPT`、导图或摘要，而是一个可生成、可引用、可协作、可演化的课程知识单元。一个库通常包含知识结构、资源索引、权限设置、生成能力入口、引用关系和版本状态。

### `Course Knowledge Hub`（课程资料库）

指围绕同一课程或项目沉淀的多模态知识空间，产品上可视为“库”的一种教学场景表达。内容可包括思维导图、PPT、教案、动画、网页、程序示例、视频和参考资料。其核心特点是可分享、可持续更新、可进行权限管理与隐私控制。

### `Project`（项目 / 空间）

指系统内部的主业务对象，也是数据隔离和资源归属的基本单位。对外可被理解为课程空间、学习空间或资料空间，也就是产品层所说的“库”；对内继续采用通用的 `project` 命名，以保证系统具备场景可迁移性。

### `Session`（工作会话）

指同一项目下独立的对话、草稿、预览和生成上下文。`Session` 用于隔离不同用户或同一用户不同轮次的工作过程，避免上下文串扰。

### `Reference`（引用）

指一个项目对另一个项目的结构化引用关系。引用可分为跟随模式与固定模式，并可进一步区分主基底引用与辅助引用。所有引用关系必须满足有向无环图（`DAG`）约束。

### `Follow`（跟随）

指当前库直接跟踪上游库的最新合法状态，不复制完整底座。上游变化后，底层知识源自动更新，但导图、摘要、`PPT` 等外化结果只在访问或重新生成时刷新。

### `Pinned`（拉取 / 固定引用）

指当前库以某个上游版本为固定起点进行独立演化。上游后续变化不会自动影响当前库，适合需要稳定起点和深度定制的场景。

### `Primary Base Reference`（主基底引用）

指一个库的主课程骨架来源。每个库最多只能拥有一个主基底引用。

### `Auxiliary Reference`（辅助引用）

指用于补充案例、题库、实验、风格模板或跨学科内容的引用来源。一个库可以拥有多个辅助引用，但默认用于补充，不覆盖主骨架。

### `Candidate Change`（候选变更）

指协作者在独立会话中提出、尚未写入正式库状态的变更建议。候选变更需要由库维护者审核后才能合入。

### `Version`（版本）

指库的正式稳定状态标识。只有正式入库的结构化变化才产生新版本；草稿、预览、对话和临时生成结果不单独记为正式版本。

### `Artifact`（派生结果 / 导出物）

指从项目当前状态按需外化出的结果，例如 `PPTX`、`DOCX`、思维导图、摘要、网页和动画资源。`Artifact` 不是系统长期沉淀的本体，而是课程数据库在特定场景下的输出结果，并应记录其来源库与来源版本。

### `Seamless Asset Accumulation`（无感化资产沉淀）

指教师在正常使用系统生成 PPT、教案和其他课程内容时，系统自动捕获教学逻辑、知识结构和素材关系，并将其自然沉淀为课程资料库。该过程不要求教师额外上传、转发或手工整理，因此是一种“生产即沉淀、沉淀即交付”的副产物机制。

### `Iterative Optimization`（迭代式优化）

指教师基于当前生成结果提出反馈，系统再依据反馈进行局部或整体调整，从而形成“互动-生成-反馈-再生成”的循环。

### `Prompt Routing`（提示词路由）

指系统根据资料类型、任务类型或教学场景，自动选择更合适的提示词模板，以提升意图理解和生成效果。

### `Vector Database`（向量数据库）

指存储文本向量或多模态向量并支持相似度检索的数据库。`Spectra` 使用 `ChromaDB` 作为向量检索基础设施。

## 项目角色术语

### `教师端`

指用于创建项目、上传资料、发起生成、预览修改和管理课程资料库的主要工作台。

### `学生端`

指通过教师邀请进入课程资料库的访问侧。学生可基于授权查看课程成果，并进一步生成自己的思维导图、复习资料或摘要内容。

### `项目`

指系统内围绕一次课程建设活动组织起来的基本单位，通常包含课程主题、对话历史、上传资料、知识库内容、生成结果和版本记录。

## 写作约束

1. 正式文档中首次出现英文缩略语时，应同时给出中文全称。
2. “库”与“课程资料库”必须与“静态文件分享”区分开来，强调课程数据库本体、动态更新与权限控制。
3. “智能体”必须体现主动追问、信息融合和再生成能力，不能退化为“调用一个聊天模型”。
4. 对外文档允许使用“课程资料库”“课程空间”“学习空间”等产品词；系统实现层统一采用 `project/session/reference/artifact` 等通用命名。
