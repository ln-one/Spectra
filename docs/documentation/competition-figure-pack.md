## 1. 总原则

### 1.1 这份文档要解决什么问题

当前主稿已经够强，最大的短板不是内容缺口，而是：

- 有些章节图太少
- 有些章节虽然有 Mermaid，但还只是骨架
- 全书还没有形成“翻开就像完整详细设计方案”的视觉完成面

所以这一轮不是继续写正文，而是把全书需要什么图、每张图回答什么问题、谁来画、画到什么程度，一次性说死。

### 1.2 配图总要求

所有正式图统一遵守：

- 每张图只回答一个问题
- 每张图脱离正文也能大致独立阅读
- 每张图都能用一句话概括其判断
- 图要帮助评审快速形成判断，而不是增加阅读负担

统一风格要求：

- 商业级方案书视觉
- 结构清晰
- 标签短
- 层次明确
- 留白足够
- 能单页独立阅读

统一禁止项：

- 赛博蓝紫渐变背景
- 装饰性图标大于结构信息
- UML / 类图作业风
- Mermaid 原样截图美化后直接交差
- 一张图同时讲三件事
- 英文缩写堆砌到不看正文就看不懂

### 1.3 章节视觉角色

整本书按章节承担不同视觉角色：

- `00 / 01 / 02 / 10`：开场与结尾判断图
- `03 / 04`：系统本体、对象语言、结构成立图
- `05`：关键机制与实现图
- `06`：成果与证据图
- `07 / 09`：治理与风险图
- `08`：商业与平台图
