# Spectra 主稿 DOCX 排版规则

> Status: `current`
> Role: formatting contract for the external commercial proposal `DOCX`.

## 1. 目标

正式主稿的排版目标不是“能导出”，而是：

- 规范
- 优雅
- 冷静
- 像中文正式提交件

它不能是：

- Pandoc 默认长文档
- 普通论文模板
- 死板公文
- 花哨海报

## 2. 文档结构

正式主稿的固定结构为：

1. 封面
2. 目录
3. 执行摘要
4. 正文第 `1` 章开始
5. 结语结束

约束：

- 执行摘要前置，但不占正文 `1`
- 正文从 [01-overview.md](/Users/ln1/Projects/Spectra/docs/competition/01-overview.md) 开始编号
- [11-a-class-project-overview.md](/Users/ln1/Projects/Spectra/docs/competition/11-a-class-project-overview.md) 不并入这份大文档

## 3. 版心与页面

- 纸张：`A4`
- 页边距：上下左右均 `2cm`
- 页码：正文统一阿拉伯数字页码
- 标题切换：一级标题自然换章，不靠手工空行

## 4. 标题体系

标题编号固定为：

- 一级标题：`1. 标题`
- 二级标题：`1.1 标题`
- 三级标题：`1.1.1 标题`

目录只反映正式正文层级，避免像 Markdown 文件列表。

## 5. 正文段落

正文统一遵守：

- 中文正文：宋体
- 西文和数字：Times New Roman
- 字号：小四等效
- 首行缩进：两个中文字符
- 行距：固定值 `22pt`
- 段后：轻量留白，不靠空行撑版
- 对齐：两端对齐

正文风格要求：

- 一段只承载一个核心判断
- 段落短而稳
- 页面要有呼吸感
- 不允许出现 Markdown 默认散排痕迹

## 6. 目录

目录必须满足：

- 自动生成
- 层级清楚
- 页码稳定
- 与标题样式强绑定

禁止：

- 手敲目录
- 目录像文件树
- 目录字体、缩进、行距混乱

## 7. 表格

全文表格统一为：

- 有线表格
- 表头加粗
- 表头背景轻量区分
- 单元格垂直居中
- 跨页重复表头

表题规则：

- 统一写作 `表 x-x`
- 放在表格上方
- 风格简洁，不写成长说明

禁止：

- `Normal Table` 默认视觉
- 无线框裸表
- 表头和正文混成一个密度

## 8. 图位

当前阶段不放正式图片时，图位必须以正式占位块出现。

统一规则：

- 图题统一为 `图 x-x`
- 图位块居中
- 图位块有明确边界和轻量底色
- 图位说明短，不写内部注释味

禁止：

- Mermaid 代码块直接进入 Word
- 图位像开发者提示
- 图题和图位断裂

## 9. 风格边界

`docs_origin/Spectra.docx` 只可借：

- 中文提交件的整体调性
- 局部标题节奏
- 页面整体感觉

不可直接继承：

- 无线框表格
- 非规范目录
- 风格混杂残留

## 10. 生产链

正式导出入口固定为：

```bash
python3 /Users/ln1/Projects/Spectra/scripts/build_competition_docx.py
```

生产方式固定为：

1. 合并 `00` 到 `10`
2. Mermaid 转规范图位占位块
3. `pandoc` 导出基础 `DOCX`
4. `python-docx` 后处理标题、段落、目录样式、表格、图题和页码

约束：

- 优先修改源 Markdown
- 不长期直接在 Word 里漂移维护
