## 术语及缩略语

为保证全文表述一致，现对本文涉及的核心对象和相关服务术语作如下说明。

### 1. 核心业务对象

| 术语 | 本文中的定义 |
| --- | --- |
| `Project` | 项目资料与结果的统一管理空间，用于承接长期存在的教学资料、成果对象及其引用关系 |
| `Session` | 一次具体的业务处理过程或会话过程，用于承接生成、修改、确认和状态推进 |
| `Artifact` | 系统生成的成果对象，如课件、文档或其他形式的内容产物 |
| `Version` | 成果对象保存后的版本记录，用于记录正式状态并支撑后续回看与比较 |
| `Reference` | 不同资料、成果对象或管理空间之间的引用关系 |
| `CandidateChange` | 进入后续修改、确认或版本演进流程的候选变更 |
| `Member` | 协作成员及其权限边界定义 |

### 2. 相关能力服务

| 术语 | 本文中的定义 |
| --- | --- |
| `Diego` | 内容生成服务，负责大纲生成、正式生成和迭代优化相关处理 |
| `Pagevra` | 预览、渲染与导出服务，负责成果查看和标准化交付 |
| `Ourograph` | 成果状态、版本与引用关系管理服务 |
| `Dualweave` | 数据上传、解析与交付处理服务 |

上述术语在全文中保留英文对象名，目的是保持系统语义的一致性和对象关系的稳定性。为保证方案书可读性，文中同时给出中文定义。后续章节首次出现相关术语时，以中文业务含义为主，必要时补充英文对象名。
