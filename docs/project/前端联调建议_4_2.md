# 前端联调建议

## 一、会话文件命名问题

### 1. 问题现状

当前联调中存在命名分叉：

- 历史记录标题与真实 PPT 工件标题不一致（出现硬编码文案，如 `PPT Ready`）。
- 部分下载命名仍使用前端兜底格式（如 `artifact-xxxx.ext`），未与工件标题保持一致。
- Word 教案虽已绑定 `source_artifact_id`，但来源 PPT 信息在界面上不够直观。

### 2. 联调目标

命名必须收口为同一语义源，确保用户看到的名称、历史记录名称、下载名称一致可追踪。

### 3. 建议约束

1. 历史记录标题：优先使用 `artifact.metadata.title`，仅在缺失时回退默认文案。
2. 下载文件名：与工件标题一致，不再前端硬编码 `artifact-xxxx.ext`。
3. Word 教案命名：按“源 PPT 标题 + 教案”生成显示与下载名（扩展名为 `.docx`）。
4. Word 来源可视化：在界面显式展示：
   - `source_artifact_id`
   - 来源 PPT 标题（由该 artifact 的 `metadata.title` 解析）

### 4. 联调验收清单（本部分）

- [ ] 生成 PPT 后，历史记录显示名称与该 PPT 的 `metadata.title` 一致。
- [ ] 下载 PPT 时，文件名与历史/工件展示名一致。
- [ ] 生成 Word 后，命名为“源 PPT 标题 + 教案.docx”。
- [ ] Word 区域可看到其绑定的 `source_artifact_id` 与来源 PPT 标题。
- [ ] 以上命名在刷新页面后仍保持一致（非一次性前端态）。

### 5. 风险提示

1. 若后端未写入 `metadata.title`，前端需明确回退策略并打日志，避免静默降级。
2. 若来源 artifact 不可访问（权限/删除），Word 来源展示需给出可理解提示，不应误导为“无来源”。

## 二、教案链路收口（直接选择对应 PPT 再生成）

### 1. 联调结论（后端已验证）

- `word_document` 主链路已收口为“必须提供 `source_artifact_id`（且类型为 `pptx`）”。
- 缺少 `source_artifact_id`：返回 `400 INVALID_INPUT`。
- `source_artifact_id` 不存在：返回 `404 NOT_FOUND`。
- `source_artifact_id` 类型非 `pptx`：返回 `400 INVALID_INPUT`。

### 2. 前端接入约束

1. 生成 Word 教案入口不再暴露松散表单，改为“从可选 PPT 列表中单选一个 source”。
2. 未选择 PPT 时，“生成教案”按钮必须禁用，并给出明确提示。
3. 仅当当前会话存在有效 Word artifact 时，“下载教案”按钮可用，禁止下载历史残留造成误判。
4. 生成成功后，界面需显式展示：
   - 当前 `source_artifact_id`
   - 来源 PPT 标题
   - 当前 Word artifact id

### 3. 联调验收清单（本部分）

- [ ] 不选择 PPT 时，前端无法触发教案生成请求。
- [ ] 选择无效 source（不存在）时，前端正确展示 404 错误语义。
- [ ] 选择非 PPT source 时，前端正确展示 400 错误语义。
- [ ] 选择有效 PPT source 时，Word 生成成功并返回新 artifact。
- [ ] 下载按钮仅绑定当前有效 Word artifact，不可误下载历史残留文件。
