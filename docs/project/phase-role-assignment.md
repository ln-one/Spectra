# 当前阶段分工（完整产品交付版）

> 阶段目标：直接交付“前端导向设计文档”对应的可用完整产品（非仅对齐阶段）。
> 本阶段必含能力补齐：MinerU/LlamaParse、Qwen-VL、Faster-Whisper。

## 一、本阶段交付定义（Done = 可用产品）

1. 用户可完整走通：配置 -> 大纲共创 -> 确认生成 -> 预览编辑 -> 局部重绘 -> 导出。
2. 关键交互不依赖 Mock，全部走真实 API。
3. 输出具备可溯源入口（至少页级来源：chunk/file/page/time）。
4. 文档解析采用可插拔方案：MinerU（主）+ LlamaParse（备）。
5. 视频理解与语音识别已接入可用链路：Qwen-VL、Faster-Whisper。

## 二、成员分工（工作包级）

## 成员 A（架构师 / Tech Lead）

工作包 A1：完整产品范围冻结
- 内容：冻结“本阶段必须可用”的功能边界，避免边做边扩。
- 产出：产品完成清单（按用户旅程，不按技术模块）。
- 完成标准：团队对“什么算上线可用”口径一致。

工作包 A2：跨能力编排与契约守护
- 内容：统一生成状态机、解析链路、视频/语音入口在同一任务编排下运行。
- 产出：跨模块接口边界文档 + 契约评审结论。
- 完成标准：不存在“前端有入口但后端无语义”的断层。

工作包 A3：上线门禁（产品级）
- 内容：建立产品级 DoD（可用性、稳定性、可恢复、可溯源）。
- 产出：看板可勾选 DoD 与上线准入清单。
- 完成标准：每个模块 PR 都能映射到产品 DoD。

## 成员 B（前端主程）

工作包 B1：完整流程页面交付
- 内容：交付前端设计文档中的核心页面与关键交互，不留“占位页”。
- 产出：页面与组件重构 PR（生成流程全链路）。
- 完成标准：业务方可直接体验完整流程。

工作包 B2：多能力入口整合
- 内容：在同一工作台接入文档、视频、语音能力的可操作入口与状态反馈。
- 产出：多模态入口与状态面板 PR。
- 完成标准：用户能明确选择与使用不同能力，不靠隐式配置。

工作包 B3：可溯源体验落地
- 内容：输出页展示来源入口，支持查看来源详情与定位信息。
- 产出：Source 面板、引用标记、空来源兜底提示 PR。
- 完成标准：用户可解释“这页内容来自哪里”。

工作包 B4：能力降级提示体验
- 内容：当 MinerU/Qwen-VL/Whisper 不可用时，展示明确降级提示与当前回退路径。
- 产出：降级提示组件与交互文案 PR。
- 完成标准：用户知道“当前用了哪个回退能力、结果可能有什么差异”。

工作包 B5：完整回归验证
- 内容：补齐端到端用例（生成、重写大纲、局部重绘、视频输入、语音输入、恢复、能力降级）。
- 产出：E2E 用例与回归报告。
- 完成标准：关键用户旅程可重复通过。

## 成员 C（后端主程 / 数据流）

工作包 C1：任务执行主链路落地
- 内容：把会话、状态机、重试、恢复做成稳定执行链路。
- 产出：任务系统与接口实现 PR。
- 完成标准：任务不中断、可恢复、可追踪。

工作包 C2：解析器可插拔落地（本阶段必做）
- 内容：为 `file_parser` 接入 MinerU/LlamaParse 策略与切换。
- 产出：解析器接口层 + 环境变量策略 + 失败回退实现 PR。
- 完成标准：三种策略可切换，失败可回退，不阻断主流程。

工作包 C3：视频/语音执行通道接入
- 内容：打通 Qwen-VL、Faster-Whisper 的调用链路与任务状态回传。
- 产出：视频理解与语音识别后端接入 PR。
- 完成标准：前端触发后能拿到可用结果并进入后续生成流程。

工作包 C4：外部能力降级策略落地
- 内容：当 MinerU/Qwen-VL/Whisper 不可用时自动回退，并返回标准化降级信息。
- 产出：降级策略实现 PR（含错误码、回退目标、用户提示语）。
- 完成标准：外部能力异常不会中断主流程，前端可据字段稳定提示。

## 成员 D（AI/RAG 工程师）

工作包 D1：解析质量与路由策略
- 内容：定义 MinerU/LlamaParse 的选路策略与质量阈值。
- 产出：解析质量基准、策略建议、异常样本集。
- 完成标准：不同文档类型可稳定命中合适解析器。

工作包 D2：视频理解能力可用化（Qwen-VL）
- 内容：定义视频理解输入输出结构、提示词模板、结果后处理。
- 产出：Qwen-VL 提示词与结果结构规范 + 验证报告。
- 完成标准：视频输入可产出可用教学内容片段。

工作包 D3：语音识别能力可用化（Faster-Whisper）
- 内容：定义音频处理、识别参数、文本清洗与错误回退。
- 产出：Whisper 参数配置与识别质量报告。
- 完成标准：语音输入可稳定转文本并进入生成流程。

工作包 D4：溯源质量校验
- 内容：建立“输出-来源映射”轻量评测，验证相关性与可解释性。
- 产出：溯源评测脚本与结果。
- 完成标准：来源信息对用户可读且基本可信。

工作包 D5：降级策略质量基线
- 内容：定义各能力降级后的质量下限与可接受范围（解析/视频/语音）。
- 产出：降级质量基线说明与抽样报告。
- 完成标准：降级后结果仍满足“可用但有差异”的明确标准。

## 三、跨成员协作依赖（必须同步）

1. A -> 全员：产品完成清单与上线门禁。
2. C <-> D：解析/视频/语音能力的输入输出契约与失败语义。
3. C -> B：真实返回样例（成功、失败、冲突、回退、无来源）。
4. D -> B：来源展示优先级与可解释字段说明。
5. D -> C：降级策略质量下限与回退优先级建议。

## 四、本阶段验收清单（产品视角）

- 完整流程可用：从素材输入到导出可走通。
- 三能力补齐：MinerU/LlamaParse、Qwen-VL、Faster-Whisper 均可用。
- 契约一致：前后端行为与 OpenAPI 一致。
- 体验可解释：关键输出可看到来源入口。
- 异常可恢复：失败、冲突、断线场景可继续操作。
- 能力可降级：外部能力不可用时主流程不断，且前端有明确提示。
