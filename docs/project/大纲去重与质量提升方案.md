# 大纲去重与质量提升方案

> 更新时间：2026-03-25
> 状态：当前生效
> 定位：成员 D 针对大纲重复问题的第一版优化方案

## 1. 目标

本方案用于解决当前大纲中的四类问题：

1. 章节标题重复
2. 关键要点重复
3. 相邻章节原地复述、没有推进
4. 标题和要点过泛，不利于后续生成

目标不是“让大纲看起来不同”，而是让每一章都承担清晰教学功能，并能稳定进入后续 PPT 生成。

## 2. 当前判定口径

当前默认使用以下指标判断大纲是否存在重复与质量问题：

1. `title_uniqueness_pass_rate`
2. `key_point_uniqueness_pass_rate`
3. `cross_section_progression_pass_rate`
4. `expression_specificity_pass_rate`
5. `overall_pass_rate`

对应脚本与基线：

1. `backend/eval/outline_quality_audit.py`
2. `backend/eval/outline_quality_baseline.py`
3. `backend/eval/baselines/outline-quality-baseline-v1.json`

## 3. 优化方向

大纲去重默认按三层推进：

1. prompt 层
   - 明确禁止重复标题、重复知识点、相邻章节同义复述
2. heuristic 层
   - 对低质量大纲做结构修复、标题去重、要点去重
3. regression 层
   - 用样本和 baseline 验证优化没有退化

## 4. 质量提升原则

1. 标题必须体现章节功能差异，而不是只写“核心知识点”“内容讲解”。
2. 关键要点必须各自承担不同讲解任务，不允许在相邻章节换说法重复。
3. 每个章节都应向后推进教学流程，而不是停留在同一层知识复述。
4. 优化后大纲应更利于后续 PPT 生成，而不是为了去重牺牲可讲性。

## 5. 下一步执行

下一步专项优化默认落在以下两处：

1. 大纲生成 prompt 约束
2. 大纲生成后的低质量修复与去重 heuristics
