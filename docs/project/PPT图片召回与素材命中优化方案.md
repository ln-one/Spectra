# PPT 图片召回与素材命中优化方案

> 更新时间：2026-03-25
> 状态：当前生效
> 定位：成员 D 针对 PPT 图片素材利用质量的第一版优化方案

## 1. 目标

当前阶段不先追求“每页都有图”，而是先解决两个问题：

1. 什么时候应该插图
2. 插入的图是否真的支撑当前页面讲解

本方案用于约束 PPT 生成中的图片素材使用，减少“弱相关装饰图”“看起来有图但讲解用不上”的情况。

## 2. 素材命中原则

1. 只在当前知识点明显需要图示、实验现象、结构图、流程图时考虑插图。
2. 优先使用项目资料或检索结果中的高相关素材，而不是泛化背景图。
3. 如果没有足够匹配的素材，默认不插图，不强行配视觉元素。
4. 图片必须能直接支撑本页标题或核心结论，不能只是主题相关但讲解无用。
5. 图片一旦使用，页面文字必须与图片形成明确对应，不允许“图文各讲各的”。

## 3. 当前命中质量判定口径

当前先按以下口径判定一张图是否命中：

1. 与页面核心标题或结论直接相关
2. 能被教师在口头讲解中明确指向
3. 不是纯装饰用途
4. 不会挤压掉本页必要文本空间

如果以上条件不能满足，应判定为“不命中”。

## 4. 当前接入方式

当前这版方案先通过两种方式落地：

1. 进入 PPT 生成 prompt 规则
2. 进入后续样本评测与人工对照口径

当前不在 D 侧直接实现真实图片插入链路；真实图片接入与导出闭环仍归主线功能侧处理。

## 5. 下一步

下一步继续收口：

1. 什么情况下必须插图、什么情况下不应插图
2. 插什么类型的图
3. 图片与页面文字、版式之间如何对应
