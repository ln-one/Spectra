# Archived Documentation

> Updated: 2026-03-21
> Status: archived

This directory preserves historical context, superseded drafts, legacy rollout plans, and one-off records.

Use archived docs only when you need:

- historical decision context
- old rollout assumptions
- comparison against earlier ontology or API ideas
- recovery of removed documentation content

Do not use archived docs as the default implementation source.

## Main Archived Buckets

- `project/`: historical project discussions and handoff material
- `project-space/`: superseded project-space design drafts
- `specs/`: legacy or superseded contracts/specs
- `plans/`: old execution plans and work packets
- `openapi/`: superseded OpenAPI guides
- `reports/`: old reports and status records

If an active document links here, it should do so explicitly as historical/reference-only context.
